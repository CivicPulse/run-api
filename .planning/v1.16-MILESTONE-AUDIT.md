1. Phase 95's provider seam is reused by invite email construction and submission.
2. Phase 96 queues invite mail only after durable invite state exists.
3. Phase 96 invite responses expose delivery-summary fields needed by support UI.
4. Phase 97 creates canonical attempt rows keyed to invite identity and provider message ID.
5. Phase 97 webhook reconciliation updates invite projections without relying on recipient email alone.
6. Mailgun webhook verification is isolated from user-authenticated APIs.
7. Invite acceptance remains same-origin and separate from ZITADEL notification ownership.
8. Phase 98 keeps ZITADEL SMTP configuration outside CivicPulse runtime code paths.
9. Phase 98 documents the ownership boundary between invite mail and auth/system mail.
10. Phase 99 consolidates DNS, sender-alignment, and monitoring expectations across both email systems.
11. Incident routing now distinguishes CivicPulse runtime failures, ZITADEL SMTP failures, and shared Mailgun/DNS failures.
12. Scope remains bounded to transactional/system mail only; no campaign-authored or bulk-email surface was introduced.
