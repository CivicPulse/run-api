<objective>
Fix docker compose up --build failure on re-bootstrap by adding a client_credentials token fallback when PAT file does not exist.

Purpose: The PAT file (/zitadel-data/pat.txt) is only written by ZITADEL on first-instance init. On subsequent boots (volume persists), the file is never created. The bootstrap script currently always calls read_pat() which waits 60s then fails. When credentials are already valid, we should obtain a bearer token via client_credentials grant instead.

Output: Updated scripts/bootstrap-zitadel.py that handles both fresh and re-bootstrap scenarios.
</objective>
