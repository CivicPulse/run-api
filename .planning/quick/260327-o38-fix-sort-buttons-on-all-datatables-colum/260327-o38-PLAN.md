Modify `DataTable.tsx` to support both server-side and client-side sorting:

1. Add `getSortedRowModel` to the import from `@tanstack/react-table`.

2. In the `useReactTable` config, make `manualSorting` conditional:
   - When `onSortingChange` IS provided: keep `manualSorting: true` (server-side mode, current behavior). Do NOT add `getSortedRowModel`.
   - When `onSortingChange` is NOT provided: set `manualSorting: false` and add `getSortedRowModel: getSortedRowModel()`. This enables automatic client-side sorting managed by TanStack Table internally.

3. The `onSortingChange` handler in useReactTable should remain as-is for the server-side case. For client-side case, omit the `onSortingChange` option entirely (let TanStack manage sorting state internally). This means the `state.sorting` should also only be set when `onSortingChange` is provided (server-side mode). For client-side mode, let TanStack Table manage its own sorting state.

Concrete implementation for the useReactTable call:
```typescript
const isManualSorting = !!onSortingChange

const table = useReactTable({
  data,
  columns,
  state: {
    ...(isManualSorting ? { sorting: sorting ?? [] } : {}),
  },
  ...(isManualSorting
    ? {
        onSortingChange: (updater) => {
          const newSorting =
            typeof updater === "function" ? updater(sorting ?? []) : updater
          onSortingChange(newSorting)
        },
        manualSorting: true,
      }
    : {
        manualSorting: false,
        getSortedRowModel: getSortedRowModel(),
      }),
  manualFiltering: true,
  manualPagination: true,
  getCoreRowModel: getCoreRowModel(),
})
```
