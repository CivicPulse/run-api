## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, badge internal padding |
| sm | 8px | Voter sub-card internal padding, gap between badge and text |
| md | 16px | Default element spacing, card padding, section gaps |
| lg | 24px | Gap between household cards, padding around wizard content |
| xl | 32px | Top/bottom page padding within field layout |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Outcome buttons use 12px (`gap-3`) between grid items for comfortable tap targets
- FieldHeader is fixed at `h-14` (56px) per existing implementation
- Progress bar sits directly below the header with 0px gap (flush)

## Interaction Specifications

### OutcomeGrid

- 2-column CSS grid: `grid grid-cols-2 gap-3`
- Each button: `min-h-11` (44px WCAG), full-width within column, `border` + category color classes
- Single tap records outcome immediately (no confirmation)
- Disabled state with `opacity-50 pointer-events-none` when voter is already completed

### VoterCard

- Renders inside HouseholdCard as a stacked sub-card
- Active voter: full card with OutcomeGrid inline below voter info
- Completed voter: compact card showing name + recorded outcome badge + checkmark
- Voter info layout: name (text-base font-normal) on first line, party badge + propensity badge + age on second line, prior interaction on third line
- All badges use `text-xs` with `px-2 py-0.5` (existing Badge sizing)

### HouseholdCard

- Outer container: `Card` with `p-4` padding
- Address header: `text-lg font-semibold` with `MapPin` icon (lucide), tappable (`cursor-pointer`)
- Tap on address opens Google Maps navigation: `https://www.google.com/maps/dir/?api=1&destination={encodedAddress}`
- Caption "Tap address to navigate" in `text-xs text-muted-foreground` below address
- Voter sub-cards separated by `Separator` component with `my-3` spacing
- "Skip" button at bottom right: `variant="ghost"` with `SkipForward` icon, `text-sm text-muted-foreground`

### InlineSurvey (Sheet Panel)

- Opens as `Sheet` with `side="bottom"`
- Max height: `max-h-[70dvh]` to leave room for mobile keyboard
- Rounded top corners: `rounded-t-2xl`
- Content scrolls vertically: `overflow-y-auto`
- Question layout: question text (`text-base`) followed by answer input
  - `multiple_choice`: radio group with `min-h-11` tap targets per option
  - `scale`: slider component
  - `free_text`: textarea with `min-h-[80px]`
- Footer pinned at bottom: "Skip Survey" (variant="ghost") left, "Save Answers" (variant="default") right
- Overlay dims card content behind: `bg-black/50` (default Sheet overlay)

### FieldProgress

- Sits directly below FieldHeader, outside scrollable content
- Text: "{current} of {total} doors" in `text-sm text-muted-foreground`, centered
- Bar: shadcn `Progress` component, `h-1` (4px thin bar), full width
- Container: `px-4 py-2 border-b`

### ResumePrompt

- Rendered as a `sonner` toast with custom content (not a banner)
- Toast content: heading + door info + two buttons + countdown
- "Resume" button: `variant="default"`, `min-h-11`
- "Start Over" button: `variant="outline"`, `min-h-11`
- Auto-dismiss after 10 seconds with visible countdown: "Resuming in {N}..."
- Toast position: `top-center` (below field header)
- Duration: 10000ms with `onAutoClose` triggering resume action

### DoorListView

- Opens as `Sheet` with `side="bottom"`, `max-h-[80dvh]`
- Header: "Walk List" title with close button
- List items: each row shows sequence number, address, status badge
- Status badges: "Pending" (outline), "Visited" (green), "Skipped" (gray)
- Tap on any row jumps wizard to that address: `onJump(addressIndex)`
- Current address highlighted with `bg-accent` background
- Scrollable list with `overflow-y-auto`
- Trigger button in wizard UI: "All Doors" with `List` icon, `variant="outline"`, placed to the right of progress text

### Auto-Advance Behavior

- After non-contact outcome: 300ms delay, then slide to next voter/address
- After survey complete/skip: immediate advance
- After bulk "Not Home" apply: 500ms delay (visual feedback of all cards updating), then advance to next address
- Transition: card slides out left, new card slides in from right (`animate-in slide-in-from-right-4` + `animate-out slide-out-to-left-4`)

## Accessibility (A11Y-04)

### ARIA Live Regions

- Hidden `div` with `aria-live="polite" aria-atomic="true" role="status" className="sr-only"` wrapping a text node
- Text updates on every door/address transition: "Door {N} of {total}. {address}. {voter name}."
- Text updates on outcome recording: "{outcome} recorded for {voter name}."
- Text updates on survey open: "Survey questions. {count} questions."
- Text updates on wizard completion: "Walk list complete. {total} doors visited."

### Touch Targets

- All outcome buttons: `min-h-11 min-w-11` (44px) — matches FieldHeader pattern
- Skip button: `min-h-11`
- Survey radio options: `min-h-11`
- Address tap area: entire address header row is tappable, `min-h-11`
- List view rows: `min-h-11` per row

### Focus Management

- On door advance: focus moves to the new household card heading
- On survey open: focus moves to Sheet content (handled by radix Sheet)
- On survey close: focus returns to the voter card that triggered it
- On resume prompt dismiss: focus moves to current household card

## State Architecture

### Zustand Persist Store (`canvassingStore.ts`)

```
canvassing-wizard (sessionStorage key)
  walkListId: string | null
  currentAddressIndex: number
  completedEntries: Record<string, string>  // entryId -> resultCode
  skippedEntries: string[]
  lastActiveAt: number  // timestamp
```

- Only navigation/completion state in Zustand (not entry data)
- Entry data lives in React Query cache (`useWalkListEntries`)
- Survey answers submitted immediately via mutation (not stored in persist)
- Stale session detection: `lastActiveAt` older than 4 hours triggers re-fetch + reconciliation
