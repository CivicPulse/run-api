### Semantic Color Additions for Phase 33

| State | Color Strategy | Rationale |
|-------|---------------|-----------|
| Offline (no signal) | `bg-muted border-b` with `text-foreground` | Neutral/muted — non-alarming; signal lost is expected in field |
| Syncing (reconnected) | Same `bg-muted` + `text-muted-foreground` + lucide `Loader2` spinner (animated) | Same surface; spinner indicates activity |
| Sync complete | Sonner toast only — banner disappears; no dedicated color | Positive closure without persistent UI |

Accent reserved for:
- Pending outcome count in OfflineBanner (the number "N" in "Offline • N outcomes saved")
- Sync spinner (`Loader2` icon, `text-primary` class)
- No other elements in Phase 33 use accent

Outcome buttons (OutcomeGrid) retain their domain-specific colors (green/yellow/red/orange per OUTCOME_COLORS map) — these are unchanged from Phase 31/32 and are NOT modified by offline state.

## OfflineBanner Interaction States

The banner is the sole new visible surface in Phase 33. All states must be implemented exactly as specified.

### State 1: Online, Queue Empty — Hidden

```
[renders null — zero DOM nodes]
```

- Condition: `isOnline === true && items.length === 0`
- Result: Component returns `null`. No DOM insertion, no height, no layout shift.

### State 2: Offline — Showing

```
┌─────────────────────────────────────────┐
│  ⚡  Offline • 3 outcomes saved          │
└─────────────────────────────────────────┘
```

- Condition: `isOnline === false`
- Height: 32px (`h-8`)
- Background: `bg-muted`
- Border: `border-b border-border`
- Text: `text-sm text-foreground` — "Offline • {N} outcomes saved"
- Icon: `WifiOff` lucide icon, 14px (`h-3.5 w-3.5`), `text-muted-foreground`
- Pending count `{N}` renders with `font-semibold` weight when N > 0
- When N = 0 (offline but nothing queued yet): "Offline"
- Padding: `px-4` (16px) horizontal; content centered vertically
- Layout: `flex items-center justify-center gap-2`
- ARIA: `role="status"` `aria-live="polite"` `aria-label="You are offline. {N} outcomes saved locally."`

### State 3: Reconnected, Syncing

```
┌─────────────────────────────────────────┐
│  ◌  Syncing 3 outcomes...               │
└─────────────────────────────────────────┘
```

- Condition: `isOnline === true && isSyncing === true`
- Height: 32px — same layout as State 2
- Background: `bg-muted` (same)
- Text: `text-sm text-muted-foreground` — "Syncing {N} outcomes..."
- Icon: `Loader2` lucide icon, 14px, `animate-spin text-primary`
- ARIA: `role="status"` `aria-live="polite"` `aria-label="Syncing {N} outcomes to server."`

### State 4: Online, Queue Draining to Empty — Toast + Disappear

When `isSyncing` transitions from `true` to `false` and `items.length === 0`:
- OfflineBanner returns `null` (disappears)
- Sonner toast fires: `toast.success("All caught up!")` — duration 3000ms
- If walk list data shows entries visited by others: secondary toast fires: `toast("5 doors were visited by others")` — duration 4000ms
- If current entry was completed by another volunteer: toast fires: `toast("This door was visited — moving to next")` — duration 3000ms, then wizard auto-advances

### State 5: Online, Items Remain After Sync Attempt (retry pending)

- Banner stays visible in State 2 style (offline-like appearance)
- Count reflects remaining un-synced items
- No user-facing explanation — retry is automatic

### Connectivity Debounce

- Going offline: banner appears immediately (0ms delay)
- Coming back online: wait 1000ms before transitioning to State 3 (prevents flicker on unstable connections)
- Implementation: `setTimeout(() => setStableOnline(true), 1000)` inside `online` event handler

## Copywriting Contract

| Element | Copy | Notes |
|---------|------|-------|
| Banner: offline, 0 queued | "Offline" | No count when nothing queued yet |
| Banner: offline, N queued | "Offline • {N} outcomes saved" | Bullet separator, count updates in real-time |
| Banner: syncing | "Syncing {N} outcomes..." | N = items.length at sync start; ellipsis indicates in-progress |
| Toast: sync complete | "All caught up!" | Sonner success toast, 3000ms |
| Toast: team progress | "{N} doors were visited by others" | Sonner info toast, 4000ms; only shown if N > 0 |
| Toast: conflict skip | "This door was visited — moving to next" | Sonner default toast, 3000ms; auto-advance follows |
| ARIA offline | "You are offline. {N} outcomes saved locally." | aria-label on banner element |
| ARIA syncing | "Syncing {N} outcomes to server." | aria-label on banner element |

No primary CTA for this phase — the sync is fully automatic. No destructive actions. No empty state (offline queue being empty = no UI shown).

## Interaction Contract Summary

| Interaction | Trigger | Response | Duration |
|-------------|---------|----------|----------|
| Record outcome while offline | Volunteer taps outcome button | Optimistic UI update persists; item queued; NO revert | Instant |
| Connectivity lost | `offline` window event | Banner appears immediately (State 2) | Instant |
| Connectivity restored | `online` window event + 1s debounce | Banner transitions to State 3 (syncing) | 1s delay |
| Sync complete | Last item removed from queue | Banner disappears; "All caught up!" toast | 3s toast |
| Team progress | Post-sync query refetch detects others' completions | Secondary toast with count | 4s toast |
| Conflict skip | Server returns 409 during sync | Item discarded silently; "This door was visited" toast; wizard auto-advances | 3s toast |
| Periodic drain | 30s interval while online | Silently drains any remaining items (no UI if already synced) | Background |

| Element | ARIA Implementation |
