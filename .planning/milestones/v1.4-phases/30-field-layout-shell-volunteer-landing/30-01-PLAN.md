  <name>Task 1: Backend field/me endpoint with service and schemas</name>
  <files>app/schemas/field.py, app/services/field.py, app/api/v1/field.py, app/api/v1/router.py, tests/unit/test_field_me.py</files>
  <behavior>
    - GET /api/v1/campaigns/{campaign_id}/field/me returns 200 with FieldMeResponse
    - Response includes volunteer_name (from user.display_name or email), campaign_name
    - Response includes canvassing: {walk_list_id, name, total, completed} or null
    - Response includes phone_banking: {session_id, name, total, completed} or null
    - Canvassing assignment is most recent by assigned_at DESC, limit 1
    - Phone banking assignment is most recent active session by created_at DESC, limit 1
    - Phone banking progress uses CallList.total_entries and CallList.completed_entries via session.call_list_id join
    - RLS context is set before queries (set_campaign_context)
    - Returns 401 if not authenticated
  </behavior>
  <action>
    1. Create `app/schemas/field.py`:
       - Import BaseSchema from app.schemas.common
       - Define CanvassingAssignment(BaseSchema): walk_list_id (uuid.UUID), name (str), total (int), completed (int)
       - Define PhoneBankingAssignment(BaseSchema): session_id (uuid.UUID), name (str), total (int), completed (int)
       - Define FieldMeResponse(BaseSchema): volunteer_name (str), campaign_name (str), canvassing (CanvassingAssignment | None), phone_banking (PhoneBankingAssignment | None)

    2. Create `app/services/field.py`:
       - Import models: WalkList, WalkListCanvasser, PhoneBankSession, SessionCaller, CallList, Campaign
       - Class FieldService with async method get_field_me(self, db: AsyncSession, campaign_id: uuid.UUID, user_id: str, display_name: str | None, email: str | None) -> dict
       - Query most recent canvassing: SELECT WalkList JOIN WalkListCanvasser ON walk_list_id WHERE user_id = user_id AND campaign_id, ORDER BY assigned_at DESC, LIMIT 1
       - Query most recent active phone banking: SELECT PhoneBankSession JOIN SessionCaller ON session_id WHERE user_id = user_id AND campaign_id AND status = "active", ORDER BY created_at DESC, LIMIT 1. Then join CallList via session.call_list_id for total_entries and completed_entries.
       - Query Campaign.name
       - Return dict matching FieldMeResponse shape

    3. Create `app/api/v1/field.py`:
       - FastAPI router with GET /campaigns/{campaign_id}/field/me
       - Dependencies: get_current_user, get_db
       - Call set_campaign_context(db, str(campaign_id))
       - Call ensure_user_synced(user, db) per project convention
       - Delegate to FieldService.get_field_me()
       - Return FieldMeResponse.model_validate(result)

    4. Wire in `app/api/v1/router.py`:
       - Import field module
       - Add: router.include_router(field.router, tags=["field"])
       - Note: field.py router already includes /campaigns/{campaign_id}/field/me path, so no prefix needed on include_router

    5. Create `tests/unit/test_field_me.py`:
       - Test service method with mocked DB session
       - Test cases: volunteer with both assignments, only canvassing, only phone banking, no assignments
       - Verify correct assignment is selected (most recent)
  </action>
