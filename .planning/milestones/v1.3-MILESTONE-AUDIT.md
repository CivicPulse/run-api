### Integration Gaps (Non-Blocking)

**INT-01 (P1 — Visual Break):** ImportJob TypeScript type misaligned with backend ImportJobResponse
- `filename` field should be `original_filename` — import history table Filename column renders blank
- `error_count` field has no backend counterpart — Errors column renders blank
- Files: `web/src/types/import-job.ts`, `web/src/routes/campaigns/$campaignId/voters/imports/index.tsx`

**INT-02 (P2 — Silent Filter):** `tags_any` filter chip missing from live pages
- `filterChipUtils.ts` handles `tags_any` correctly in `buildStaticChipDescriptors`
- But `buildFilterChips` (voter list) and `buildDialogChips` (dynamic list dialogs) omit it
- Filter is sent to backend but invisible to user and not dismissible via chip
- Files: `web/src/routes/.../voters/index.tsx`, `web/src/routes/.../voters/lists/index.tsx`

**INT-03 (P3 — Missing Control):** `registration_county` filter has no UI control
- Backend: `build_voter_query` condition at `app/services/voter.py` exists and tested
- Frontend: VoterFilterBuilder Location section has city/state/zip but no county input
- Files: `web/src/components/voters/VoterFilterBuilder.tsx`

**Type Safety Note:** `VoterSearchBody.sort_by` typed as `string` instead of union of 12 valid column names. No runtime impact (backend validates and returns 422 for invalid values), but degrades TypeScript type checking.
