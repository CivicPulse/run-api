1. **app/services/voter_interaction.py** — VoterInteractionService:

   **record_interaction(session, voter_id, interaction_type, payload, user_id) -> VoterInteraction**
   - Create VoterInteraction record with provided fields
   - Set created_at explicitly (same pattern as User sync in Phase 1 — don't rely on server_default)
   - NEVER issue UPDATE or DELETE on VoterInteraction table
   - Return the created record

   **get_voter_history(session, voter_id, type_filter=None, cursor=None, limit=50) -> PaginatedResponse**
   - Query voter_interactions WHERE voter_id, ordered by created_at DESC
   - Optional filter by interaction type
   - Cursor pagination on (created_at, id)

   **record_correction(session, original_event_id, correction_payload, user_id) -> VoterInteraction**
   - Create a new interaction with type="correction" (add CORRECTION to InteractionType enum if not present, or use NOTE type with payload referencing original)
   - payload includes `original_event_id` reference
   - This is how "corrections" work — new event, not mutation

   Generic service handling all three contact types (phone, email, address). Use a helper pattern or separate methods per type.

   **add_phone(session, voter_id, value, type, is_primary, source, user_id) -> VoterPhone**
   - If is_primary=True, set all other phones for this voter to is_primary=False
   - Create VoterPhone record
   - Emit interaction event: `record_interaction(session, voter_id, "contact_updated", {"action": "added", "contact_type": "phone", "value": value}, user_id)`

   **add_email(session, voter_id, value, type, is_primary, source, user_id) -> VoterEmail**
   - Same primary flag management and interaction event pattern

   **add_address(session, voter_id, address_data, type, is_primary, source, user_id) -> VoterAddress**
   - Same pattern but with structured address fields

   **update_phone/email/address** — Update existing contact, emit contact_updated event
   **delete_phone/email/address** — Delete contact record, emit contact_updated event with action "removed"
   **get_voter_contacts(session, voter_id) -> dict** — Return all contacts grouped by type: {"phones": [...], "emails": [...], "addresses": [...]}
   **set_primary(session, contact_type, contact_id, voter_id)** — Set one contact as primary, unset others of same type

1. **app/api/v1/voter_interactions.py** — Interaction endpoints:

   **GET /api/v1/campaigns/{campaign_id}/voters/{voter_id}/interactions** — Get voter history
   - Query params: type (optional filter), cursor, limit
   - Returns PaginatedResponse with interaction events, newest first
   - Requires: `require_role("volunteer")`

   **POST /api/v1/campaigns/{campaign_id}/voters/{voter_id}/interactions** — Add interaction (manual note)
   - Accepts: `{ type: str = "note", payload: dict }`
   - Only allows "note" type via API (other types created by system operations)
   - Returns interaction record
   - Requires: `require_role("volunteer")`

2. **app/api/v1/voter_contacts.py** — Contact management endpoints:

   **GET /api/v1/campaigns/{campaign_id}/voters/{voter_id}/contacts** — Get all contacts
   - Returns: `{ phones: [...], emails: [...], addresses: [...] }`
   - Requires: `require_role("volunteer")`

   **POST /api/v1/campaigns/{campaign_id}/voters/{voter_id}/phones** — Add phone
   - Accepts PhoneCreate
   - Returns PhoneResponse
   - Requires: `require_role("manager")`

   **PATCH /api/v1/campaigns/{campaign_id}/voters/{voter_id}/phones/{phone_id}** — Update phone
   - Requires: `require_role("manager")`

   **DELETE /api/v1/campaigns/{campaign_id}/voters/{voter_id}/phones/{phone_id}** — Delete phone
   - Returns 204
   - Requires: `require_role("manager")`

   **POST/PATCH/DELETE for emails** — Same pattern as phones

   **POST/PATCH/DELETE for addresses** — Same pattern, using AddressCreate/AddressResponse

   **POST /api/v1/campaigns/{campaign_id}/voters/{voter_id}/contacts/{contact_type}/{contact_id}/set-primary** — Set primary
   - contact_type: "phones", "emails", or "addresses"
   - Returns 204
   - Requires: `require_role("manager")`
