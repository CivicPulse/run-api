  <action>
Create `app/services/import_service.py` with ImportService class:

1. **CANONICAL_FIELDS dict** — Map each canonical voter field to a list of known aliases (as shown in research). Include L2-specific aliases (LALVOTERID, Voters_FirstName, Voters_LastName, Parties_Description, Residence_Addresses_AddressLine, etc.).

2. **suggest_field_mapping(csv_columns: list[str]) -> dict[str, str | None]** — Use RapidFuzz `process.extractOne` with `fuzz.ratio` scorer and 75 score_cutoff. Normalize column names to lowercase with underscores. Return dict mapping each CSV column to canonical field name or None.

3. **detect_columns(file_content: bytes) -> list[str]** — Read first line of CSV using csv.reader with encoding detection. Try UTF-8 first, fall back to latin-1. Return list of column headers.

4. **process_csv_batch(rows: list[dict], field_mapping: dict, campaign_id: str, source_type: str, session: AsyncSession) -> tuple[int, list[dict]]** — For each row, apply field mapping to produce voter dict. Mapped fields go to proper columns, unmapped fields go to extra_data. Use PostgreSQL `INSERT ... ON CONFLICT (campaign_id, source_type, source_id) DO UPDATE` via SQLAlchemy `insert().on_conflict_do_update()`. Return (imported_count, error_list).

5. **process_import_file(import_job_id: str, session: AsyncSession, storage: StorageService)** — Main orchestration: download file from S3 via storage.download_file(), stream through csv.DictReader with batch size of 1000 rows, call process_csv_batch for each batch, update ImportJob status/counts after each batch, generate error report CSV and upload to S3 if errors exist, set final status to completed/failed.

CRITICAL: In background task context, call `set_campaign_context(session, campaign_id)` before any DB operations (research pitfall #3).

Write tests first (RED), then implement (GREEN):
- `tests/unit/test_field_mapping.py` — Test suggest_field_mapping with various inputs: standard names, L2 names, unknown columns, case insensitivity, partial matches
- `tests/unit/test_import_service.py` — Test detect_columns, process_csv_batch with mock session (test mapping logic, skip logic, extra_data handling)

1. **app/tasks/import_task.py** — Background import task:
   ```python
   @broker.task
   async def process_import(import_job_id: str) -> dict:
   ```
   - Create a new async DB session (not from request context — this runs in background worker)
   - Load ImportJob by ID, get campaign_id
   - Call `set_campaign_context(session, str(campaign_id))` — CRITICAL for RLS
   - Update status to PROCESSING
   - Delegate to `ImportService.process_import_file()`
   - Update ImportJob with final counts and status (COMPLETED or FAILED)
   - Handle exceptions: catch broadly, set status=FAILED with error_message, re-raise
   - Use `from __future__ import annotations`

2. **app/api/v1/imports.py** — Import workflow endpoints (all require admin+ role):

   **POST /api/v1/campaigns/{campaign_id}/imports** — Step 1: Initiate import
   - Accepts: `{ original_filename: str, source_type: str = "csv" }`
   - Creates ImportJob with status=PENDING
   - Generates S3 key: `imports/{campaign_id}/{import_job_id}/{original_filename}`
   - Calls StorageService.generate_upload_url(key)
   - Returns ImportUploadResponse with import_job_id + upload_url
   - Requires: `require_role("admin")`

   **POST /api/v1/campaigns/{campaign_id}/imports/{import_id}/detect** — Step 1b: After upload, detect columns
   - Downloads first few KB of file from S3
   - Calls ImportService.detect_columns() and suggest_field_mapping()
   - Updates ImportJob: status=UPLOADED, detected_columns, suggested_mapping
   - Returns detected columns + suggested mappings
   - Optionally accepts `template_id` query param to load a saved template mapping instead of auto-suggest

   **POST /api/v1/campaigns/{campaign_id}/imports/{import_id}/confirm** — Step 2: Confirm mapping and start
   - Accepts: ConfirmMappingRequest with `field_mapping: dict, save_template: bool = False, template_name: str | None`
   - Updates ImportJob: field_mapping, status=QUEUED
   - If save_template: create FieldMappingTemplate record
   - Dispatches TaskIQ job: `await process_import.kiq(str(import_id))`
   - Returns ImportJobResponse with status=QUEUED

   **GET /api/v1/campaigns/{campaign_id}/imports/{import_id}** — Poll status
   - Returns ImportJobResponse with current status, counts, error_report_url (pre-signed GET URL if error_report_key exists)
   - Requires: `require_role("admin")`

   **GET /api/v1/campaigns/{campaign_id}/imports** — List import jobs
   - Returns paginated list of ImportJobResponse for the campaign
   - Requires: `require_role("admin")`

   **GET /api/v1/campaigns/{campaign_id}/imports/templates** — List mapping templates
   - Returns system templates (is_system=true, campaign_id IS NULL) + campaign templates
   - Requires: `require_role("admin")`

3. **app/api/v1/router.py** — Add imports router:
   ```python
   from app.api.v1 import imports
   router.include_router(imports.router, tags=["imports"])
   ```

All endpoints use RLS via set_campaign_context. Access StorageService from `request.app.state.storage_service`. Follow RFC 9457 problem details for errors. Use Google-style docstrings.
