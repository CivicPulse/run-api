       - `app/models/campaign.py`: Campaign model with fields: id (UUID PK, default uuid4), zitadel_org_id (str, unique, indexed), name (str), type (Enum: federal/state/local/ballot), jurisdiction_fips (str, nullable), jurisdiction_name (str, nullable), election_date (date, nullable), status (Enum: active/suspended/archived/deleted, default active), candidate_name (str, nullable), party_affiliation (str, nullable), created_by (UUID FK to users.id), created_at, updated_at. Use `mapped_column` style per CONVENTIONS.md.
       - `app/models/user.py`: User model with fields: id (str PK -- ZITADEL subject UUID as string), display_name (str), email (str), created_at, updated_at. The id is the ZITADEL `sub` claim, stored as string not UUID.
       - `app/models/campaign_member.py`: CampaignMember with fields: id (UUID PK), user_id (str FK to users.id), campaign_id (UUID FK to campaigns.id), synced_at (datetime). NO role column -- role comes from JWT. Unique constraint on (user_id, campaign_id).

    6. **Create Pydantic schemas:**
       - `app/schemas/common.py`: BaseSchema (with `from_attributes=True`), PaginationResponse (next_cursor: str|None, has_more: bool), PaginatedResponse generic wrapper.

Add lifespan handler that initializes JWKSManager on app.state. Replace the old `main.py` at project root with an import: `from app.main import create_app; app = create_app()`.

       - Run `uv run alembic init alembic`
       - Configure `alembic/env.py` to use async engine (asyncpg) and import Base.metadata from app.db.base. Use `config.get_main_option("sqlalchemy.url")` for the connection string. Support both sync (psycopg2) for offline and async for online migrations.
       - In `alembic.ini`, set `sqlalchemy.url` to use env var override pattern.
       - Create initial migration `001_initial_schema.py` that:
         a. Creates `users`, `campaigns`, `campaign_members` tables
         b. Creates `app_user` database role: `CREATE ROLE app_user WITH LOGIN PASSWORD 'app_password' NOINHERIT`
         c. Enables RLS on all three tables: `ALTER TABLE {table} ENABLE ROW LEVEL SECURITY`
         d. Creates RLS policies:
            - `campaigns`: `USING (id = current_setting('app.current_campaign_id', true)::uuid)`
            - `campaign_members`: `USING (campaign_id = current_setting('app.current_campaign_id', true)::uuid)`
            - `users`: Access via campaign_members join: `USING (id IN (SELECT user_id FROM campaign_members WHERE campaign_id = current_setting('app.current_campaign_id', true)::uuid))`
         e. Grants SELECT/INSERT/UPDATE/DELETE on all tables to app_user
         f. Sets default privileges for future tables
         Note: Use `current_setting('app.current_campaign_id', true)` (second param = true) to return NULL on missing instead of error (RESEARCH.md Open Question 3).

       - `AuthenticatedUser` dataclass/Pydantic model: id (str), org_id (str), role (CampaignRole), email (str|None), display_name (str|None).
       - `CampaignRole(IntEnum)`: VIEWER=0, VOLUNTEER=1, MANAGER=2, ADMIN=3, OWNER=4.
       - `JWKSManager` class: __init__(issuer), async get_jwks(force_refresh=False), async validate_token(token) -> dict. Uses Authlib `jwt.decode()` with `JsonWebKey.import_key_set()`. On decode failure with unknown kid, retry with force_refresh=True. Cache JWKS indefinitely until failure (per RESEARCH.md discretion).

Returns `AuthenticatedUser`. Raises HTTPException 401 on any validation failure.
