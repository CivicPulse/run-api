**VolunteerService (app/services/volunteer.py):**
Follow CampaignService/CanvassService patterns. Methods:

- `create_volunteer(session, campaign_id, data, created_by)` -- Create volunteer record. If user_id provided, set it.
- `self_register(session, campaign_id, user_id, data)` -- Logged-in user self-registers. Auto-link user_id, set created_by=user_id. Check for existing volunteer with same user_id in campaign (409 if exists).
- `get_volunteer(session, volunteer_id)` -- Fetch by ID.
- `get_volunteer_detail(session, volunteer_id)` -- Fetch with tags and availability included.
- `update_volunteer(session, volunteer_id, data)` -- Partial update.
- `update_status(session, volunteer_id, new_status)` -- Enforce transitions: pending->active, active->inactive, pending->inactive only. No backward transitions.
- `list_volunteers(session, campaign_id, filters, cursor, limit)` -- Filterable list. Filters: status, skills (any match on ARRAY), name search (ILIKE on first_name || last_name), tag_ids (via join to volunteer_tag_members). Cursor pagination on (created_at, id).
- `add_tag(session, volunteer_id, tag_id)` -- Add tag to volunteer.
- `remove_tag(session, volunteer_id, tag_id)` -- Remove tag.
- `create_tag(session, campaign_id, name)` -- Create campaign-scoped tag.
- `list_tags(session, campaign_id)` -- List campaign tags.
- `add_availability(session, volunteer_id, data)` -- Add availability slot. Validate end_at > start_at.
- `delete_availability(session, availability_id)` -- Remove slot.
- `list_availability(session, volunteer_id)` -- List volunteer's availability slots.
- `find_available_volunteers(session, campaign_id, start_at, end_at)` -- Find active volunteers with availability covering the given window.

**ShiftService (app/services/shift.py):**
Follow PhoneBankService composition pattern. Compose with no direct service imports at module level (use late imports for WalkListCanvasser/SessionCaller to avoid circular deps).

Methods:
- `create_shift(session, campaign_id, data, created_by)` -- Create shift. Validate: if type=CANVASSING, turf_id should be set; if type=PHONE_BANKING, phone_bank_session_id should be set. These are optional per CONTEXT but emit warning log if missing.
- `get_shift(session, shift_id)` -- Fetch shift with signed_up_count and waitlist_count computed via subquery on shift_volunteers.
- `update_shift(session, shift_id, data)` -- Partial update. Only allowed when status is SCHEDULED.
- `update_status(session, shift_id, new_status)` -- Enforce transitions: scheduled->active, scheduled->cancelled, active->completed. Cancelled preserves signups.
- `list_shifts(session, campaign_id, filters, cursor, limit)` -- Filter by status, type, date range. Cursor pagination on (start_at, id).
- `delete_shift(session, shift_id)` -- Only if status=SCHEDULED and no checked-in volunteers.
- `signup_volunteer(session, shift_id, volunteer_id)` -- Core signup logic:
  1. Verify volunteer status is ACTIVE (pending blocked per CONTEXT)
  2. If shift.type in (CANVASSING, PHONE_BANKING): verify volunteer has emergency_contact_name and emergency_contact_phone (raise 422 if missing)
  3. Check not already signed up (409 if exists)
  4. Count current signed_up volunteers. If < max_volunteers: status=SIGNED_UP. If >= max_volunteers: status=WAITLISTED, assign waitlist_position = max existing position + 1.
  5. Return ShiftVolunteer record.
- `cancel_signup(session, shift_id, volunteer_id, requester_id, requester_role)` -- Self-cancel: only if shift.start_at > now(). Manager+: any time. Set status=CANCELLED. Call `_promote_from_waitlist`.
- `manager_assign(session, shift_id, volunteer_id)` -- Manager assigns volunteer (bypasses capacity check). Still enforces emergency contact for field shifts and active-status requirement.
- `_promote_from_waitlist(session, shift_id)` -- SELECT ... FOR UPDATE the next WAITLISTED entry ordered by waitlist_position, promote to SIGNED_UP, clear waitlist_position.
- `check_in(session, shift_id, volunteer_id)` -- Set check_in_at=now(), status=CHECKED_IN. Side effects:
  - If shift.type=CANVASSING and shift.turf_id: resolve walk list (most recent for turf via `select(WalkList.id).where(WalkList.turf_id == shift.turf_id).order_by(WalkList.created_at.desc()).limit(1)`). Create WalkListCanvasser(walk_list_id, user_id=volunteer.user_id). If volunteer has no user_id, skip WalkListCanvasser creation (walk-in volunteers without accounts cannot be canvassers -- log warning).
  - If shift.type=PHONE_BANKING and shift.phone_bank_session_id: create SessionCaller(session_id=shift.phone_bank_session_id, user_id=volunteer.user_id, check_in_at=now()). Same user_id requirement.
  - Use late imports inside this method to avoid circular deps.
- `check_out(session, shift_id, volunteer_id)` -- Set check_out_at=now(), status=CHECKED_OUT. If phone banking shift and SessionCaller exists, also set SessionCaller.check_out_at.
- `adjust_hours(session, shift_id, volunteer_id, adjusted_hours, reason, adjusted_by)` -- Set adjusted_hours, adjustment_reason, adjusted_by, adjusted_at=now(). Original check_in/out preserved.
- `get_volunteer_hours(session, volunteer_id, campaign_id)` -- Aggregate hours query per RESEARCH.md: SUM of CASE(adjusted_hours if not null, else epoch diff / 3600). Return total_hours, shifts_worked.
