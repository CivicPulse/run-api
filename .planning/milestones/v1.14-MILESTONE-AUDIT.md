## Cross-Phase Integration (10/10 Wired)

The milestone wiring is clean across the main seams:

1. Voter-page lookup input writes into the existing `filters.search` field instead of bypassing the search contract.
2. Search requests continue through `POST /voters/search`, keeping structured filters and lookup semantics in one flow.
3. Default lookup ordering uses a dedicated search cursor payload so ranked pagination stays stable.
4. Search execution routes through campaign-scoped `voter_search_records` rather than raw cross-field joins.
5. Voter, contact, and import writes refresh the same search surface across both backend refresh paths and frontend query invalidation.
6. Ranked search spans names, contacts, addresses, city, ZIP, and source identifiers from the denormalized surface.
7. Typo-tolerant token matching composes with the ranking buckets rather than replacing deterministic filters.
8. Query hooks preserve prior data and propagate abort signals so stale responses do not win the race.
9. Contact edits, including address deletion, invalidate voter queries so refreshed search data reaches the voter page immediately.
10. Backend and frontend tests still exercise the main lookup contract shape, including the repaired address-delete seam.

## E2E User Flows (4/4 Complete)
