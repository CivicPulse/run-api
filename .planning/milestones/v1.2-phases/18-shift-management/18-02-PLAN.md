1. Create `web/src/components/shifts/ShiftDialog.tsx`:

   - react-hook-form with zod schema validation:
     - name: required, 3-100 chars
     - type: required, one of SHIFT_TYPES
     - start_at: required, ISO datetime string
     - end_at: required, ISO datetime string, must be after start_at
     - max_volunteers: required, number >= 1
     - description: optional string
     - location_name: optional string
     - street, city, state, zip_code: optional strings
     - turf_id: optional string (dropdown populated from `useTurfs(campaignId)`)
     - phone_bank_session_id: optional string (dropdown populated from `usePhoneBankSessions(campaignId)`)
   - Use `<Input type="datetime-local" />` for start_at and end_at (project standard, per SessionDialog pattern)
   - Wire useFormGuard({ form }) for unsaved changes protection
   - On submit: call `useCreateShift` (create mode) or `useUpdateShift` (edit mode)
   - On success: toast("Shift created"/"Shift updated"), close sheet
   - On error: toast.error("Failed to create/update shift")
   - Title: "Create Shift" or "Edit Shift"
   - In edit mode, default form values from editShift fields. Convert editShift.start_at/end_at ISO strings to datetime-local format (slice first 16 chars of ISO string)

   - Volunteer self-action buttons:
     - Show "Sign Up" button when shift.status === "scheduled" (handles "already signed up" 422 error gracefully -- show toast "Already signed up" and no crash)
     - Show "Join Waitlist" when shift is full (cannot determine from list counts -- show generic "Sign Up" for v1 since list endpoint returns 0 counts; detail page handles accurate counts)
     - The practical v1 approach: show a "Sign Up" button on all scheduled shifts. When useSelfSignup returns 422 "already signed up", show toast.      - Do NOT show Cancel on list cards since we cannot know current user's signup status from list endpoint. Cancel is available on the detail page.
   - Use `useDeleteShift(campaignId)` for delete action
   - Use `useUpdateShiftStatus(campaignId, shift.id)` for status change actions
   - Use `useSelfSignup(campaignId, shift.id)` for sign up button
   - ConfirmDialog for status transitions (Activate, Mark Complete, Cancel Shift)

   - Format time range: extract hours/minutes from start_at and end_at, display as "9:00 AM - 12:00 PM" style
