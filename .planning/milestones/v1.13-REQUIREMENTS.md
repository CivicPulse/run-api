# Requirements: CivicPulse Run API

**Defined:** 2026-04-05
**Core Value:** Any candidate, regardless of party or budget, can run professional-grade field operations from a single API.

## v1.13 Requirements

Requirements for milestone `v1.13 Production Shakedown Remediation`. Source of truth for scope: `docs/production-shakedown/results/`.

### Tenant Isolation

- [ ] **ISO-01**: List-member and call-list body foreign keys must be validated against the path campaign so cross-tenant body injection attempts create no rows.
- [ ] **ISO-02**: Voter interactions and other transitive tenant-scoped resource references must enforce campaign ownership before read or write operations proceed.
- [ ] **ISO-03**: Turf voter spatial queries must return only voters belonging to the path campaign, even for large legacy datasets and join-heavy queries.
- [ ] **ISO-04**: Volunteer and shift-related UUID lookups must be scoped to the path campaign for detail, hours, status, and availability operations.
- [ ] **ISO-05**: `/field/me` and related field bootstrap endpoints must require campaign membership and leak no foreign campaign metadata.

### Errors & Security

- [ ] **ERR-01**: Database, validation, and race-condition failures must be normalized into safe API errors without exposing stack traces, SQL, constraints, or internal infrastructure details.
- [ ] **SEC-01**: Production responses must include CSP, `X-Frame-Options`, and `X-Content-Type-Options` headers appropriate for the deployed SPA/API stack.
- [ ] **SEC-02**: Plain HTTP traffic must redirect to HTTPS, and HTTPS responses must include HSTS.

### Operational Workflow Recovery

- [ ] **OPS-01**: Campaign creation must remain successful and idempotent when the required ZITADEL grant already exists.
- [ ] **OPS-02**: CSV import confirm/pre-scan worker flow must complete successfully instead of failing immediately in the worker.
- [ ] **OPS-03**: Volunteer signup and cancellation flows must have stable, documented idempotent behavior.
- [ ] **OPS-04**: Phone-banking session/result APIs must reject nonexistent `call_list_id` values and invalid `result_code` values with client-safe validation errors.
- [ ] **OPS-05**: Survey question reorder must reject partial ID sets and preserve valid full reorder behavior.

### Accessibility & Field Launch Readiness

- [ ] **UI-01**: Reported unlabeled `SelectTrigger` and icon-only controls must expose accessible names on the affected routes.
- [ ] **UI-02**: Stretched-link overlays and similar routed card affordances must expose accessible link names.
- [ ] **UI-03**: Field/mobile controls and volunteer field-entry routing must satisfy the intended launch UX standard, including target size and correct entry point behavior.
- [ ] **PERF-01**: Field hub mobile cold-load must meet the launch target or be explicitly rebaselined with fresh evidence and documentation.

### Validation, Semantics & Documentation

- [ ] **VAL-01**: Remaining request-shape and input-validation edge cases from the shakedown must reject invalid inputs safely and consistently.
- [ ] **VAL-02**: Lifecycle and deleted-state behaviors that differed from plan expectations must be either corrected in code or explicitly documented as the supported contract.
- [ ] **VAL-03**: Remaining survey, phone-bank, and field data-integrity gaps must be resolved by code change or explicit product decision.
- [ ] **VAL-04**: Remaining production-surface, semantics, and observability gaps from the shakedown must be explicitly decided and documented.

### Verification & Cleanup

- [ ] **VRF-01**: Targeted shakedown reruns must prove all former P0 and P1 findings are closed.
- [ ] **VRF-02**: Temporary shakedown-created data, scripts, tokens, and QA residue must be removed or intentionally archived with documentation.

## Future / Deferred

### Backlog

- **BACKLOG-01**: Zitadel major-version upgrade (`Phase 999.1`) remains parked unless remediation work proves it is required for the v1.13 fixes.
- **BACKLOG-02**: Larger platform expansion work stays deferred until the production shakedown rerun reaches launchable status.
