   - `${PROD_URL}` — Production app URL. Example: `https://run.civpulse.org`
   - `${ZITADEL_URL}` — Production ZITADEL URL. Example: `https://auth.civpulse.org`
   - `${ZITADEL_PAT}` — Production ZITADEL admin Personal Access Token
   - `${DB_HOST}` — Production PostgreSQL host. Example: `postgresql.civpulse-infra.svc.cluster.local`
   - `${DB_USER}` — Production database username
   - `${DB_PASS}` — Production database password
   - `${DB_NAME}` — Production database name
   - `${S3_ENDPOINT_URL}` — Cloudflare R2 endpoint URL
   - `${S3_ACCESS_KEY_ID}` — R2 access key
   - `${S3_SECRET_ACCESS_KEY}` — R2 secret key
   - `${S3_BUCKET}` — R2 bucket name. Example: `voter-imports`
   - `${TEST_DATA_CSV}` — Path to L2 test CSV. Example: `data/example-2026-02-24.csv`
   - `${OWNER1_PASSWORD}` — Test user password. Example: `Owner1234!`
   Include a note: "Set all variables before proceeding. Any unset variable will cause test failures."

3. **Section 0: Deployment Health Checks** (per D-11) — Pre-flight checks before any testing. Include exact curl commands:

   - HEALTH-05: Frontend load — `curl -sf "${PROD_URL}" | grep -i "civicpulse"` expect HTML with CivicPulse title
   - HEALTH-06: S3/R2 bucket access — attempt to list bucket using AWS CLI or curl with S3 auth. If fails, note import tests should be marked SKIP.
   Include a "STOP" gate: "If any HEALTH check fails (except HEALTH-06), do not proceed. Resolve the infrastructure issue first."

   ZITADEL_EXTERNAL_PORT="443" \
   ZITADEL_EXTERNAL_SECURE="true" \
   ZITADEL_URL="${ZITADEL_URL}" \
   PAT_PATH="/tmp/prod-pat.txt" \
   uv run python scripts/create-e2e-users.py
   rm /tmp/prod-pat.txt
   ```
   Include the full 15-user table (3 per role: owner, admin, manager, volunteer, viewer) with email addresses and default passwords from the script.
   Include a prerequisite note about production ZITADEL org login policy needing MFA set to "not enforced" for the test org (per Phase 60 lesson in STATE.md).

     uv run python scripts/seed.py
     ```
