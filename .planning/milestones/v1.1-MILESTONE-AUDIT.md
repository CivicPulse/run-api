## Notable Design Observations

1. **CI sed updates both image lines** — init container and main container always stay on the same SHA, preventing migration/code version mismatches.
2. **ArgoCD self-exclusion** — `argocd-app.yaml` is in `k8s/` but targets `argocd` namespace. Requires one-time manual `kubectl apply -n argocd` as documented.
3. **Alembic migration consistency** — identical `python -m alembic upgrade head` in both dev-entrypoint and K8s init container, with `DATABASE_URL_SYNC` provided by compose env and K8s secret respectively.
