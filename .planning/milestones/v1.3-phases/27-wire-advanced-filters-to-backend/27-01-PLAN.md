1. In `app/services/voter.py`, add two module-level helper functions before VoterService class:
   - `encode_cursor(item, sort_by: str | None) -> str`: Encodes `sort_column_value|id`. For datetime values, use `.isoformat()`. For None values, encode as literal string `"None"`. For all other types, use `str(val)`.
   - `decode_cursor(cursor: str, sort_by: str | None) -> tuple`: Splits on `|` (maxsplit=1). Parses cursor_id as `uuid.UUID`. For the value part: if sort_by is `created_at` or `updated_at`, parse as `datetime.fromisoformat()`. If sort_by is `age`, `propensity_general`, `propensity_primary`, `propensity_combined`, `household_size`, parse as `int()`. If value is literal `"None"`, return `None`. Otherwise return as string. Add necessary imports (`uuid`, `datetime` are already imported).

2. Update `search_voters` method signature to accept `sort_by: str | None = None` and `sort_dir: str | None = None`.

3. Replace the hardcoded `order_by(Voter.created_at.desc(), Voter.id.desc())` with dynamic sorting:
   - Determine sort column: `sort_col = getattr(Voter, sort_by) if sort_by else Voter.created_at`
   - Determine direction: `is_desc = (sort_dir or "desc") == "desc"`
   - Apply ordering with NULLS LAST for ascending: `sort_col.asc().nullslast()` or `sort_col.desc()` for descending (PostgreSQL default puts NULLs first for DESC which is acceptable). Use `.nullslast()` for ASC to push NULLs to end.
   - Always include secondary sort on `Voter.id` in same direction for stable ordering.

4. Replace hardcoded cursor logic with dynamic decode:
   - Use `decode_cursor(cursor, sort_by)` to get `(cursor_val, cursor_id)`
   - Handle NULL cursor_val: when cursor_val is None, use `sort_col.is_(None) & (Voter.id < cursor_id)` for desc, or skip NULLs already passed for asc
   - For non-NULL: use the existing `(sort_col < cursor_val) | ((sort_col == cursor_val) & (Voter.id < cursor_id))` pattern for desc, flip for asc

5. Replace hardcoded next_cursor encoding with `encode_cursor(last_item, sort_by)`.
