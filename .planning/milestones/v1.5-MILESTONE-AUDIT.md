### Integration Issues Found

**INT-01: Phase 42 turf endpoints bypass centralized RLS dependency (Low)**

`GET /turfs/overlaps` and `GET /turfs/{id}/voters` in `app/api/v1/turfs.py` use `Depends(get_db)` with inline `set_campaign_context` instead of `Depends(get_campaign_db)`. Added after Phase 39 migration completed. RLS context is set correctly at SQL level, but the architecture violates DATA-03's centralization guarantee — a future regression in these endpoints would not be caught by the pool-checkout defense that `get_campaign_db` chains.

**INT-02: Rate limiting not applied to org/turf/voter endpoints (Low)**

`@limiter.limit` decorators only applied to `join.py` endpoints. Phase 40 built the rate limiting infrastructure (tiers, per-user key function) but SlowAPI requires per-endpoint opt-in. OBS-03/OBS-04 infrastructure is complete but deployment is partial.

**FLOW-01: No connected E2E journey spec (Low)**

Individual specs cover: org dashboard, turf creation, voter search, phone bank, volunteer signup. Campaign creation wizard form submission (POST to `/api/v1/campaigns` and redirect to new campaign dashboard) has no dedicated E2E test. Steps work independently but are not chained.
