1. **app/services/storage.py** -- Add two new methods to the StorageService class (per RESL-05, D-04, D-05). Place them after the existing `delete_object` method:

   ```python
   async def list_objects(self, prefix: str) -> list[str]:
       """List object keys matching a prefix.

       Uses S3 list_objects_v2 with automatic pagination to handle
       any number of per-batch error files.

       Args:
           prefix: S3 key prefix to match (e.g. "imports/{cid}/{jid}/errors/").

       Returns:
           List of matching object keys, sorted by key name.
       """
       keys: list[str] = []
       async with self.session.client(**self._client_kwargs()) as s3:
           paginator = s3.get_paginator("list_objects_v2")
           async for page in paginator.paginate(
               Bucket=settings.s3_bucket, Prefix=prefix
           ):
               for obj in page.get("Contents", []):
                   keys.append(obj["Key"])
       return sorted(keys)

   async def delete_objects(self, keys: list[str]) -> None:
       """Delete multiple objects by key.

       Uses S3 delete_objects batch API (up to 1000 keys per call).
       No-op if keys is empty.

       Args:
           keys: List of S3 object keys to delete.
       """
       if not keys:
           return
       async with self.session.client(**self._client_kwargs()) as s3:
           # S3 delete_objects supports up to 1000 keys per call
           for i in range(0, len(keys), 1000):
               batch = keys[i : i + 1000]
               await s3.delete_objects(
                   Bucket=settings.s3_bucket,
                   Delete={"Objects": [{"Key": k} for k in batch]},
               )
   ```

2. **app/db/rls.py** -- Add `commit_and_restore_rls` helper function after the existing `set_campaign_context` function (per RESL-01, RESL-02). This encapsulates the critical pattern of committing and immediately restoring RLS context to prevent the silent failure mode where RLS resets on COMMIT:

   ```python
   async def commit_and_restore_rls(
       session: AsyncSession, campaign_id: str
   ) -> None:
       """Commit the current transaction and restore RLS context.

       PostgreSQL set_config(..., true) is transaction-scoped and resets
       on COMMIT. This helper ensures RLS is always restored after commit,
       preventing the silent failure mode where subsequent queries return
       zero rows or insert into the wrong campaign context.

       Args:
           session: The async database session.
           campaign_id: The campaign UUID to restore as RLS context.
       """
       await session.commit()
       await set_campaign_context(session, campaign_id)
   ```
