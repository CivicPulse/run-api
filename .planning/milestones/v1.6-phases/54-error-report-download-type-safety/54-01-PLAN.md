    ```tsx

      <a
        href={`/api/v1/campaigns/${campaignId}/imports/${jobId}/error-report`}
        className="text-sm text-primary underline"

      >
        Download error report
    ```

    **With:**
    ```tsx
    {jobQuery.data.error_report_key && (
      <a
        href={jobQuery.data.error_report_key}
        className="text-sm text-primary underline"
        target="_blank"
        rel="noopener noreferrer"
      >
        Download error report
      </a>
    )}
    ```

    **Why each change:**
    - `href={jobQuery.data.error_report_key}`: The `error_report_key` field is NOT a raw S3 key -- the `get_import_status` endpoint (imports.py:367-370) already replaces it with a pre-signed MinIO download URL via `storage.generate_download_url()`. Use it directly.
    - Remove `download` attribute: Per HTML spec, the `download` attribute is ignored for cross-origin URLs. MinIO pre-signed URLs are cross-origin (different port in dev, different domain in prod). The attribute has no effect.
    - Add `target="_blank" rel="noopener noreferrer"`: Opens in a new tab so wizard state is preserved. The pre-signed URL points to MinIO (different origin), not the app server.
