    Per D-11 (lock): reuse existing `_is_trusted_proxy` from `app/core/rate_limit.py`.

    In app/core/middleware/request_logging.py, add import near the top:
    ```python
    from app.core.rate_limit import _is_trusted_proxy
    ```

    Replace the client_ip extraction block at lines 81-85:

    Current:
    ```python
    client_ip = (
        _get_header(headers, b"cf-connecting-ip")
        or _get_header(headers, b"x-real-ip")
        or scope.get("client", ("",))[0]
    )
    ```

    New:
    ```python
    scope_client = scope.get("client") or ("",)
    scope_client_ip = scope_client[0] if scope_client else ""
    if scope_client_ip and _is_trusted_proxy(scope_client_ip):
        client_ip = (
            _get_header(headers, b"cf-connecting-ip")
            or _get_header(headers, b"x-real-ip")
            or scope_client_ip
        )
    else:
        client_ip = scope_client_ip
    ```

    Preserve the header precedence (cf-connecting-ip first) from the original code so rate-limit and logging agree on which header to trust.

    Note: `_is_trusted_proxy` is prefixed with `_` but is module-level and already imported by other modules implicitly (it's the natural home for this check).
