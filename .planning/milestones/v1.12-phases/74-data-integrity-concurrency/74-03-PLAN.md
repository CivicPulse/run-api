<objective>
Add compensating-transaction handling to both ZITADEL-touching code paths that can leave orphaned role grants on DB commit failure: `accept_invite` (service) and `transfer_ownership` (route handler). Both follow the existing pattern established in `app/services/join.py:199-225`.

Purpose: Closes C11 / DATA-03. Without this, a failed DB commit after successful ZITADEL role mutations leaves the external system in an inconsistent state (user has roles they shouldn't, or lost roles they should keep). Compensating rollback restores ZITADEL to its pre-transaction state.

Output: Both code paths wrapped in try/except around `db.commit()`, with inverse ZITADEL calls in the exception branch. Unit tests verify compensation fires when commit fails.
</objective>

Fill in the compensation block in REVERSE order (undo op 4, then 3, then 2, then 1). Each wrapped in its own try/except so one failure doesn't abort others:

```python
try:
    await db.commit()
except Exception as commit_exc:
    logger.error(
        "DB commit failed for transfer_ownership (campaign_id={}): {}",
        campaign_id, commit_exc,
    )
    await db.rollback()
    # Inverse of op 4: remove owner from target
    try:
        await zitadel.remove_project_role(
            settings.zitadel_project_id, target_user_id, "owner",
            org_id=zitadel_org_id,
        )
    except Exception as e:
        logger.error("Compensation step 1 (remove owner from target) failed: {}", e)
    # Inverse of op 3: re-grant target_old_role to target
    try:
        await zitadel.assign_project_role(
            settings.zitadel_project_id, target_user_id, target_old_role,
            org_id=zitadel_org_id,
        )
    except Exception as e:
        logger.error("Compensation step 2 (restore target old role) failed: {}", e)
    # Inverse of op 2: remove admin from current user
    try:
        await zitadel.remove_project_role(
            settings.zitadel_project_id, current_user_id, "admin",
            org_id=zitadel_org_id,
        )
    except Exception as e:
        logger.error("Compensation step 3 (remove admin from current) failed: {}", e)
    # Inverse of op 1: re-grant owner to current user
    try:
        await zitadel.assign_project_role(
            settings.zitadel_project_id, current_user_id, "owner",
            org_id=zitadel_org_id,
        )
    except Exception as e:
        logger.error("Compensation step 4 (restore current owner) failed: {}", e)
    raise
```
