api hooks:
- beforeRequest: reads useAuthStore.getState().getAccessToken(), sets "Authorization: Bearer <token>" if non-null
- afterResponse: on 401, clears authStore via setState({ user: null, isAuthenticated: false }) and throws AuthenticationError; on 403 throws PermissionError

api retry config: limit 3, retries on [408, 413, 429, 500, 502, 503, 504]. Tests must account for retries (either set retry:false in a test-specific client or use non-retried status codes like 401/403 which are NOT in retry list).
