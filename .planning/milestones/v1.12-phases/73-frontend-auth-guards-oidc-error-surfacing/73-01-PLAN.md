1) web/e2e/auth-guard-redirect.spec.ts (runs with NO auth state — fresh context per login.spec.ts pattern)
   - Use `test.use({ storageState: { cookies: [], origins: [] } })` to force unauthenticated.
   - Test A: "unauthenticated user hitting /campaigns/new is redirected to /login?redirect=/campaigns/new"
     - page.goto("/campaigns/new")
     - await expect(page).toHaveURL(/\/login\?redirect=/)
     - await expect(page).toHaveURL(/redirect=%2Fcampaigns%2Fnew|redirect=\/campaigns\/new/)
   - Test B: "unauthenticated user hitting deep route preserves query string in redirect param"
     - page.goto("/campaigns/some-id/settings/general?tab=danger")
     - await expect(page).toHaveURL(/\/login\?redirect=/)
     - expect URL contains the encoded original path (including settings/general)
   - Test C (smoke): "unauthenticated user on /login does NOT redirect (public route)"
     - page.goto("/login"); await expect(page).toHaveURL(/\/login/)

2) web/e2e/oidc-error.spec.ts (unauthenticated context)
   - Use `test.use({ storageState: { cookies: [], origins: [] } })`.
   - Test A: "callback URL with error and error_description renders destructive Alert"
     - page.goto("/callback?error=access_denied&error_description=User+cancelled+the+request")
     - await expect(page.getByRole("alert")).toBeVisible()
     - await expect(page.getByText(/Sign-in failed/i)).toBeVisible()
     - await expect(page.getByText(/User cancelled the request/i)).toBeVisible()
     - await expect(page.getByRole("button", { name: /Back to login/i })).toBeVisible()
   - Test B: "clicking Back to login navigates to /login"
     - After visiting callback with error, click the Back to login button
     - await expect(page).toHaveURL(/\/login/)
   - Test C: "callback with only error code (no description) still shows Alert"
     - page.goto("/callback?error=server_error")
     - await expect(page.getByRole("alert")).toBeVisible()
     - await expect(page.getByText(/server_error/)).toBeVisible()
