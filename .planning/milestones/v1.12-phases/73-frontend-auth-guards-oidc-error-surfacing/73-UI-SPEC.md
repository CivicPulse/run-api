## Component Inventory

### 1. OIDC Error State (net-new, inside `callback.tsx`)

**Trigger:** `validateSearch` extracts `error` and/or `error_description` from the OIDC callback URL before `handleCallback` is called. When either is present, render this state instead of the "Completing sign in..." spinner.

**Layout:** Full-screen centered column, matching the existing `/callback` and `/login` shell:
```
flex h-svh items-center justify-center
```

**Container:** `max-w-sm w-full mx-auto px-4`

**Structure (top to bottom):**
1. `Alert` with `variant="destructive"` — shadcn `alert.tsx` already exists
   - Icon: `AlertCircle` from lucide-react (16px, inherits destructive color)
   - `AlertTitle`: "Sign-in failed"
   - `AlertDescription`:
     - Line 1: `error_description` value (IdP human-readable text), displayed at `text-sm`
     - Line 2 (conditional, shown only when `error` code is present and differs from description): `Error code: {error}` at `text-sm text-muted-foreground`
2. `Button` with `variant="default"` (primary, civic blue), `className="w-full mt-4"`
   - Label: "Back to login"
   - On click: `navigate({ to: "/login" })` — clears the error state by navigating away

**No toast.** No auto-redirect. User must explicitly act.

**Accessibility:**
- `Alert` renders with `role="alert"` (already present in shadcn `alert.tsx`) — screen readers announce immediately on mount
- Button receives focus on mount via `autoFocus` prop — keyboard users land directly on the action
- WCAG 2.1 AA contrast: destructive border on card background meets 4.5:1 minimum; civic blue button on white background meets 7:1 at AAA level

**Error display rules:**
- If `error_description` is present: show it prominently as the primary message
- If only `error` is present (no description): show `error` as the primary message with label "Sign-in error:"
- If both are empty strings and `code`/`state` are also missing: show generic fallback — "Something went wrong during sign-in. Please try again."
- `error_description` is always the user-facing primary text; `error` code is secondary/diagnostic

### 2. Login Page — Redirect Transparency

**Trigger:** When the auth guard redirects to `/login?redirect=<original-path>`, the login page must pass the `redirect` param through `login()` so the IdP returns to `/callback` with state that resolves back to the intended URL.

**Visual change:** None. The existing "Redirecting to login..." spinner and copy are correct and unchanged.

**UX note:** No "Redirecting you to your destination..." message is needed — the redirect happens at the OIDC provider and back to `/callback`, which then navigates instantly. Users will not see intermediate state long enough to require explanation.

### 3. Role Gate Redirects (H23, H24, H25)

**When `RequireOrgRole` or `RequireRole` denies access:**

Current behavior: the `fallback` prop defaults to `null` — the page renders empty.

**Required behavior for page-level gates:** When these components wrap an entire page route, the fallback must redirect rather than silently render nothing. Two implementation approaches are in scope — the executor chooses the most consistent one with existing patterns:

Option A: Use `redirect` (TanStack Router `beforeLoad` / `loader`) — does not render the page at all.
Option B: Pass a `<Navigate to="/" />` as the `fallback` prop at each call site.

Either approach produces: **unauthorized user is immediately redirected to `/`** (the campaign list / home route). No forbidden page, no error toast. Silent redirect preserves clean UX.

**Visual contract:** No new "403 Forbidden" page. No new error component. The redirect is silent and instant.

### 4. Check-in Enforcement on Active Calling Page (H26)

**No visual change.** The existing check-in flow UI is unchanged. The enforcement moves server-side. If the server returns a non-checked-in status, the existing client-side redirect to the check-in page is invoked — no new UI state needed.

## Interaction States

| State | Component | Behavior |
|-------|-----------|----------|
| OIDC error present | `/callback` | Render error Alert + CTA instead of spinner |
| OIDC success | `/callback` | Existing spinner → navigate (unchanged) |
| Unauthenticated on protected route | Root layout | `<Navigate to="/login?redirect={pathname}" />` — instant, no flash |
| Auth loading (initialized=false) | Root layout | Existing `<Loader2>` spinner (unchanged) |
| Unauthorized role on gated page | `RequireOrgRole` / `RequireRole` | `<Navigate to="/" />` — silent redirect |
| Not checked in (calling page) | `/sessions/$sessionId/call` | Existing redirect to check-in (behavior unchanged, source of truth moves to server) |
