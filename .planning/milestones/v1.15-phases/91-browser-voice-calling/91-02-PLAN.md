<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client -> Token endpoint | Authenticated user requests Access Token; token scoped to identity + org TwiML App |
| Twilio -> TwiML handler | Twilio POSTs to our public URL; signature validated via verify_twilio_signature |
| Twilio -> Webhook handler | Status callbacks validated same way; idempotent on CallSid |
| Client -> TwiML (indirect) | Client sends To number via device.connect params; server validates in TwiML handler |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-91-04 | Spoofing | POST /voice/twiml | mitigate | verify_twilio_signature validates X-Twilio-Signature HMAC against public webhook URL |
| T-91-05 | Tampering | To number in TwiML request | mitigate | Server-side DNC check and calling hours validation in TwiML handler before dialing |
| T-91-06 | Elevation of Privilege | Token endpoint | mitigate | Requires authenticated user with campaign membership; token identity bound to user_id |
| T-91-07 | Denial of Service | Token endpoint abuse | mitigate | Rate limited to 10/minute per user; TwiML route 60/minute per IP |
| T-91-08 | Information Disclosure | Access Token JWT | accept | Token TTL 600s, contains identity and grant but no secrets; intercepted token allows calls only to numbers the TwiML handler validates |
| T-91-09 | Tampering | CampaignId param from client | mitigate | TwiML handler validates campaign membership for the caller identity before proceeding |
</threat_model>
