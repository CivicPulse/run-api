Three async functions (usable as FastAPI dependencies or called directly):

```python
async def resolve_org_from_phone(
    request: Request,
    db: AsyncSession = Depends(get_db),
) -> Organization:
```
- Read form data via `await request.form()`
- Extract `To` or `Called` param (try `To` first, fall back to `Called`)
- If neither present, raise `HTTPException(404, detail="No target number in request")`
- Query `select(OrgPhoneNumber).where(OrgPhoneNumber.phone_number == to_number)`
- If no result, raise `HTTPException(404, detail="Unknown phone number")`
- Load org via `await db.get(Organization, phone_record.org_id)`
- If org is None, raise `HTTPException(404, detail="Organization not found")`
- Return org

```python
async def verify_twilio_signature(
    request: Request,
    org: Organization = Depends(resolve_org_from_phone),
) -> Organization:
```
- Get `X-Twilio-Signature` header; if missing, raise `HTTPException(403, detail="Missing Twilio signature")`
- Reconstruct public URL: `settings.webhook_base_url.rstrip("/") + request.url.path`
- If `request.url.query`, append `"?" + request.url.query`
- Call `TwilioConfigService().credentials_for_org(org)`; if None, raise `HTTPException(403, detail="Twilio not configured for organization")`
- Create `RequestValidator(creds.auth_token)`
- Read form data: `form_data = await request.form(); params = dict(form_data)`
- Call `validator.validate(public_url, params, signature)`; if False, raise `HTTPException(403, detail="Invalid Twilio signature")`
- Return org (so downstream handlers have the resolved org)

```python
async def check_idempotency(
    db: AsyncSession,
    provider_sid: str,
    event_type: str,
    org_id: uuid.UUID,
    payload_summary: dict | None = None,
) -> bool:
```
- Use `from sqlalchemy.dialects.postgresql import insert as pg_insert`
- Build: `stmt = pg_insert(WebhookEvent).values(provider_sid=provider_sid, event_type=event_type, org_id=org_id, payload_summary=payload_summary).on_conflict_do_nothing(constraint="uq_webhook_events_sid_type")`
- Execute: `result = await db.execute(stmt)`
- Return `result.rowcount == 0` (True = duplicate, False = new event)

Import all needed types at the top. Use `from __future__ import annotations`.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet -> Webhook endpoint | Unauthenticated HTTP POST from Twilio (or attacker) crosses into the application |
| Webhook handler -> Database | Validated webhook data written to org-scoped tables |
| Phone number -> Org resolution | Inbound number determines which org's auth token and data are accessed |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-90-04 | Spoofing | Webhook endpoint | mitigate | `verify_twilio_signature` dependency validates X-Twilio-Signature HMAC-SHA1 using org-specific auth token via `RequestValidator.validate()` |
| T-90-05 | Tampering | Webhook URL | mitigate | URL reconstructed from server-side `settings.webhook_base_url`, never from `request.url`; prevents attacker-controlled URL bypass |
| T-90-06 | Repudiation | Webhook events | mitigate | `webhook_events` table logs provider_sid, event_type, org_id, and created_at for audit trail |
| T-90-07 | Information Disclosure | Cross-org data | mitigate | Phone number -> org resolution isolates auth token lookup per org; unique phone_number index ensures 1:1 mapping |
| T-90-08 | Denial of Service | Webhook flooding | mitigate | IP-based rate limiting at 120/minute via slowapi + `get_real_ip` |
| T-90-09 | Tampering | Replay/duplicate webhooks | mitigate | `check_idempotency` with UNIQUE(provider_sid, event_type) + ON CONFLICT DO NOTHING; duplicates return 200 (no retry loop) |
| T-90-10 | Elevation of Privilege | Unauthenticated webhook routes | accept | Webhook routes are intentionally unauthenticated (no JWT) -- Twilio signature validation replaces JWT auth; rate limiting provides DoS protection |
</threat_model>
