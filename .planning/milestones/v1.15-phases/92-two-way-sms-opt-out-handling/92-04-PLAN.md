## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator-facing inbox UI | Must not reveal conversations outside the selected campaign |
| Composer gating | Must not allow accidental sends when the backend reports blocked eligibility |
| Bulk-send UI | Must accurately reflect that work is queued, not silently sent inline |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-92-07 | Information Disclosure | inbox route | mitigate | use only campaign-scoped APIs and selected-conversation data |
| T-92-09 | Tampering | composer state | mitigate | disable send when `reason_code` is blocking and trust server revalidation on submit |
| T-92-10 | Repudiation | bulk-send UI | mitigate | show queued/progress states from server payloads rather than optimistic "sent" copy |
</threat_model>
