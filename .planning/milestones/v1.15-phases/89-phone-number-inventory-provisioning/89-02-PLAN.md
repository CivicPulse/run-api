<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client -> API | Untrusted browser requests; role gates enforce org_admin (read) / org_owner (write) |
| API -> Twilio | Outbound API calls; errors must be sanitized before returning to client |
| org A -> org B | Cross-org isolation enforced by filtering on org_id from JWT |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-89-01 | Information Disclosure | numbers_router endpoints | mitigate | All 5 endpoints resolve org via `Organization.zitadel_org_id == user.org_id` from JWT. Service methods filter by org_id. No cross-org query path exists. Unit tests in TestRoleGates verify isolation. |
| T-89-02 | Information Disclosure | Twilio error responses | mitigate | TwilioRestException messages are caught and replaced with generic "Twilio credential error" or "Twilio API error" strings. Raw Twilio error messages (which may contain account info) are never forwarded to the client. |
| T-89-03 | Tampering | RegisterPhoneNumberRequest | mitigate | E.164 regex validation at Pydantic schema level rejects malformed input before any DB or API call. Twilio API lookup validates number exists in the org's own account (authenticated by their credentials). |
| T-89-04 | Denial of Service | register, sync, delete endpoints | mitigate | Rate limited at 120/minute for all mutation endpoints. Twilio calls wrapped in asyncio.to_thread to prevent event loop blocking under load. |
</threat_model>
