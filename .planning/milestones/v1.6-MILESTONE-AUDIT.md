- Link `href` points to `/api/v1/.../error-report` which does not exist as a backend route
- Fix: use `href={jobQuery.data.error_report_key}` — the pre-signed MinIO URL is already available from the polling response

**INT-02: ImportUploadResponse TS type mismatch** (no runtime impact)

- Backend returns `job_id`, `upload_url`, `file_key`

- No runtime breakage — wizard only reads `job_id` and `upload_url`
