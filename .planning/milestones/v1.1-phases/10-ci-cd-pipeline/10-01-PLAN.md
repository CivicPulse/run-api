**Part B: Create `.github/workflows/publish.yml`**

Triggers on:
- `push` to `main` branch
- `push` of tags matching `v*`

Permissions: `contents: write` (for git commit-back), `packages: write` (for GHCR push).

Single job **build-and-push**:

1. Checkout code with `fetch-depth: 0` (needed for commit-back)
2. Set up Docker Buildx (`docker/setup-buildx-action@v3`)
3. Log in to GHCR (`docker/login-action@v3` with `registry: ghcr.io`, `username: ${{ github.actor }}`, `password: ${{ secrets.GITHUB_TOKEN }}`)
4. Compute tags:
   - Always: `ghcr.io/civicpulse/run-api:sha-${{ github.sha }}` and `ghcr.io/civicpulse/run-api:latest`
   - If tag push (github.ref_type == 'tag'): additionally `ghcr.io/civicpulse/run-api:${{ github.ref_name }}`
   - Use `docker/metadata-action@v5` with:
     ```yaml
     tags: |
       type=sha,format=long
       type=raw,value=latest
       type=ref,event=tag
     ```
5. Build and push (`docker/build-push-action@v6`):
   - `push: true`
   - `tags: ${{ steps.meta.outputs.tags }}`
   - `labels: ${{ steps.meta.outputs.labels }}`
   - `build-args: |`
     - `GIT_SHA=${{ github.sha }}`
     - `BUILD_TIMESTAMP=${{ steps.timestamp.outputs.value }}`
   - Use GitHub Actions cache: `cache-from: type=gha`, `cache-to: type=gha,mode=max`
6. Generate timestamp before build: `echo "value=$(date -u +%Y-%m-%dT%H:%M:%SZ)" >> $GITHUB_OUTPUT`
7. After successful push, update K8s manifest:
   ```yaml
   - name: Update K8s manifest
     run: |
       sed -i "s|image: ghcr.io/civicpulse/run-api:.*|image: ghcr.io/civicpulse/run-api:sha-${{ github.sha }}|" k8s/deployment.yaml
   ```
8. Commit and push the manifest change:
   ```yaml
   - name: Commit manifest update
     run: |
       git config user.name "github-actions[bot]"
       git config user.email "github-actions[bot]@users.noreply.github.com"
       git add k8s/deployment.yaml
       git diff --staged --quiet || git commit -m "ci: update deployment image to sha-${{ github.sha }}"
       git push
   ```
   The `git diff --staged --quiet ||` prevents empty commits if image tag unchanged.
   Commits from GITHUB_TOKEN do NOT re-trigger workflows (preventing infinite loops per user decision).

Important: The commit-back step should only run on push to main (not on tag pushes) to avoid conflicts. Add `if: github.ref == 'refs/heads/main'` to the manifest update and commit steps.
