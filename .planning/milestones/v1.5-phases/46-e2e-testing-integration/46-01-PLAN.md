   - Test "successful login redirects to campaign page":
     a. Create new context via `browser.newContext()` (no stored state)
     b. Navigate to `/`
     c. Expect ZITADEL redirect (URL contains `/loginname` or ZITADEL domain)
     d. Fill login name `admin@localhost`, click Next
     e. Fill password `Admin1234!`, click Next
     f. Expect redirect back to app with URL containing `/campaigns/` or `/org/`
     g. Expect page to contain user display name or campaign name
   - Test "invalid credentials show error":
     a. Create new context
     b. Navigate to `/`, fill login name `admin@localhost`, click Next
     c. Fill wrong password `WrongPassword123!`, click Next
     d. Expect error message visible on ZITADEL login page
   - Test "unauthenticated access redirects to login":
     a. Create new context
     b. Navigate to a protected route like `/campaigns`
     c. Expect URL to contain ZITADEL domain (redirect to login)

2. Create `web/e2e/voter-search.spec.ts`:
   - Uses stored auth state (runs in chromium project with storageState)
   - Test "search voters by name returns results":
     a. Navigate to campaign voter list page (find URL from seed data campaign)
     b. Wait for voter table/list to load (expect at least one voter row visible)
     c. Type a known seed voter name fragment into search input
     d. Wait for search results to update
     e. Expect matching voter name(s) visible in results
   - Test "filter voters by party shows filtered results":
     a. Navigate to voter list
     b. Open filter controls
     c. Apply a party filter (use a party value from seed data)
     d. Expect filtered results (fewer rows than unfiltered)
   - Test "view voter detail shows contact info":
     a. Navigate to voter list
     b. Click on a voter row to navigate to detail page
     c. Expect voter name, address, or phone visible on detail page
     d. Navigate back to list

3. Create `web/e2e/voter-import.spec.ts`:
   - Uses stored auth state
   - Test "import CSV file creates voters":
     a. Navigate to voter import page
     b. Create a small test CSV file with 3 voters using `page.evaluate` to create a Blob, or use Playwright `setInputFiles` with a buffer
     c. Upload the CSV file via the file input
     d. Map CSV columns to voter fields in the import wizard
     e. Confirm import
     f. Wait for success indication (toast, redirect, or success message)
     g. Navigate to voter list and verify imported voters appear
   - The CSV content should be: `first_name,last_name,party\nTest1,Import1,DEM\nTest2,Import2,REP\nTest3,Import3,IND`
