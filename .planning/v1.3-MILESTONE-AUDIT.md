## Critical Integration Gaps

### BREAK 1: Advanced Filters Not Reaching Backend

**Root Cause:** The voter list page uses `useVotersQuery` → `GET /campaigns/{id}/voters` which declares only 7 query params. All 12 new Phase 25 filter fields (propensity ranges, demographic multi-select, mailing address) are silently dropped by the GET endpoint.

**Affected Requirements:** FILT-01, FILT-02, FILT-03, FILT-04, FRNT-02

**Fix Path:** The backend `POST /campaigns/{id}/voters/search` endpoint already accepts a full `VoterFilter` JSON body with all 32 fields. `useSearchVoters` exists in `useVoters.ts` but is never called from the voter list page. Either:
- A) Route the voter list page through `POST /voters/search` when advanced filters are active
- B) Expand the GET endpoint's declared query params to include all filter fields

### GAP 1: Missing Filter Chips for New Filter Types

**Root Cause:** `buildFilterChips()` in `voters/index.tsx` only generates dismissible chips for legacy filter types. No chip entries for propensity, ethnicity, language, military, mailing filters.

**Affected Requirements:** FRNT-02

### GAP 2: phones_created Missing from Frontend Type

**Root Cause:** Phase 24 added `phones_created` to `ImportJobResponse` but the frontend `ImportJob` TypeScript interface was not updated.

**Affected Requirements:** IMPT-01 (display only — backend is fully wired)
