## Copywriting Contract

### Tour Step Copy

#### Welcome Segment (4 steps on hub)

| Step | Title | Description |
|------|-------|-------------|
| 1 of 4 | Welcome! | This is your home base. You'll find all your assignments right here. |
| 2 of 4 | Your Assignment | Tap your assignment to get started. We'll walk you through everything! |
| 3 of 4 | Need Help? | Tap here anytime to replay this walkthrough. |
| 4 of 4 | Your Account | Manage your account and sign out from here. |

#### Canvassing Segment (5 steps on wizard)

| Step | Title | Description |
|------|-------|-------------|
| 1 of 5 | Your First House | Here's who lives here. Check the name and address before you knock. |
| 2 of 5 | Record the Result | Tap a button after each door. Not home? Refused? Just tap and move on. |
| 3 of 5 | Track Your Progress | See how far along you are. Your progress saves automatically. |
| 4 of 5 | Jump to Any Door | Open the list to skip ahead or revisit a previous door. |
| 5 of 5 | Skip a Door | Can't reach this house? Skip it and come back later. |

#### Phone Banking Segment (3 steps on calling view)

| Step | Title | Description |
|------|-------|-------------|
| 1 of 3 | Tap to Call | Tap a phone number to open your dialer. Long-press to copy instead. |
| 2 of 3 | Record the Result | Same as canvassing -- tap a button after each call. |
| 3 of 3 | End Your Session | All done? Tap here to wrap up and see your results. |

### Quick-Start Card Copy

| Activity | Bullets |
|----------|---------|
| Canvassing | Tap a result after each door / Skip houses you can't reach / Your progress saves automatically |
| Phone banking | Tap a number to call / Record the result after hanging up / End session when you're done |

### Contextual Tooltip Copy

| Location | Popover Text |
|----------|-------------|
| Outcome buttons area | Each button records what happened at this door. "Not Home" means no one answered. |
| Progress bar | Shows doors completed out of your total. Skipped doors don't count as complete. |
| Skip button | Skipped doors stay on your list. You can come back to them anytime. |
| Phone number / tap-to-call | Tap a number to call. If your phone doesn't support tap-to-call, long-press to copy it. |

### Standard UI Copy

| Element | Copy |
|---------|------|
| Primary CTA | Next (tour advance button) |
| Final CTA | Done (last step of each segment) |
| Skip action | Skip (exits tour early; no confirmation needed) |
| Dismiss action | X icon on quick-start card (no label, aria-label: "Dismiss quick start tips") |
| Tour completion toast | Tour complete! Tap the help button anytime to replay. |
| Empty state heading | n/a (no empty states in this phase -- tour targets existing content) |
| Empty state body | n/a |
| Error state | n/a (tour is non-critical; if elements are missing, driver.js skips that step silently) |
| Destructive confirmation | n/a (no destructive actions in this phase) |

## Interaction Contracts

### driver.js Configuration

| Property | Value | Rationale |
|----------|-------|-----------|
| `overlayColor` | `"black"` | Standard dimming; matches shadcn neutral theme |
| `overlayOpacity` | `0.5` | Visible focus without feeling heavy on mobile |
| `stagePadding` | `10` | Breathing room around highlighted element |
| `stageRadius` | `8` | Matches `--radius` (0.625rem ~ 10px); 8px is close enough |
| `popoverOffset` | `12` | Space between highlight and popover arrow |
| `animate` | `true` | Smooth transitions between steps |
| `showProgress` | `true` | "Step 2 of 5" indicator per CONTEXT.md |
| `progressText` | `"Step {{current}} of {{total}}"` | Matches CONTEXT.md requirement |
| `nextBtnText` | `"Next"` | Clear forward action |
| `doneBtnText` | `"Done"` | Final step label |
| `showButtons` | `["next", "close"]` | Next to advance, close to skip/exit |
| `allowClose` | `true` | Volunteers can dismiss at any time |

### Touch Targets

| Element | Minimum Size | Implementation |
|---------|-------------|----------------|
| driver.js Next/Done button | 44x44px | CSS override: `min-height: 44px` on `.driver-popover-footer button` |
| driver.js Close/Skip button | 44x44px | CSS override: `min-height: 44px; min-width: 44px` |
| Quick-start dismiss (X) button | 32x32px (min-h-8 min-w-8) | Within a card, not primary action; 32px acceptable |
| Tooltip HelpCircle icon | 28x28px tap area | `p-1` padding on 14px icon = 22px visible, but tap area should be at least 28px via `min-h-7 min-w-7` |
| Help button in FieldHeader | 44x44px | Already `min-h-11 min-w-11` (44px) |

### Tour Trigger Behavior

| Trigger | Condition | Timing |
|---------|-----------|--------|
| Welcome auto-start | First visit to `/field/{campaignId}` AND `welcome` segment not complete | After data loads (assignment cards in DOM) + 200ms settle |
| Canvassing auto-start | First visit to `/field/{campaignId}/canvassing` AND `canvassing` segment not complete | After walk list data loads (HouseholdCard in DOM) + 200ms settle |
| Phone banking auto-start | First visit to `/field/{campaignId}/phone-banking` AND `phoneBanking` segment not complete | After caller data loads (PhoneNumberList in DOM) + 200ms settle |
| Help button replay | Tap help button on any field screen | Immediate; runs segment matching current route regardless of completion state |

### Quick-Start Card Behavior

| Behavior | Rule |
|----------|------|
| Visibility | Shows for first 3 sessions of each activity type |
| Session counting | Incremented on route mount; tracked per `{campaignId}_{userId}` per activity |
| Dismiss | X button hides for current session only (does not affect session count) |
| Tour conflict | Card does NOT render while `isRunning === true` in tour store |
| Position | Above first content element (voter card / household card), below FieldHeader |

### Tooltip Popover Behavior

| Behavior | Rule |
|----------|------|
| Trigger | Tap HelpCircle icon to open |
| Dismiss | Tap anywhere outside popover to close |
| Positioning | `side` varies by location: "top" for outcome area and progress bar, "right" for skip button, "top" for phone numbers |
| Max width | 240px |
| Always visible | Tooltip icons render regardless of tour completion state |
| During tour | Tooltip icons remain visible but non-interactive while tour overlay is active (overlay covers them) |

## CSS Override Strategy

driver.js ships its own CSS that must coexist with Tailwind v4's layer system.

| Concern | Strategy |
|---------|----------|
| Import order | `@import "driver.js/dist/driver.css"` at top of `tour.css`; `tour.css` imported in `useTour.ts` |
| Specificity | Overrides use doubled class selectors (`.driver-popover.driver-popover`) to beat driver.js defaults without `!important` |
| Font inheritance | `.driver-popover { font-family: inherit }` to use system font stack |
| Border radius | `.driver-popover { border-radius: 12px }` matching `--radius` scale |
| Popover max-width | `320px` to fit mobile viewport with margin |
| Button touch targets | All `.driver-popover-footer button` get `min-height: 44px` |
