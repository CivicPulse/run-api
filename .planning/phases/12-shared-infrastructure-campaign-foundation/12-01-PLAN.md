    - useFormGuard accepts a react-hook-form UseFormReturn instance
    - Returns { isDirty, isBlocked, proceed, reset }
    - isBlocked is true when navigation is attempted while form isDirty
    - isBlocked is false when form is clean
    - proceed() allows the blocked navigation to continue
    - reset() cancels the blocked navigation and stays on the page
    - Browser beforeunload event is enabled when form isDirty
  </behavior>
  <action>
    1. Create `web/src/hooks/useFormGuard.ts`:
       - Import `useBlocker` from `@tanstack/react-router`.
       - Accept `{ form: UseFormReturn<any> }` as options.
       - Read `form.formState.isDirty` (do NOT destructure early -- must read through Proxy for subscription).
       - Call `useBlocker({ shouldBlockFn: () => isDirty, withResolver: true, enableBeforeUnload: () => isDirty })`.
       - Return `{ isDirty, isBlocked: status === "blocked", proceed: proceed!, reset: reset! }`.
       - Per user decision: this hook wires up both route blocking AND beforeunload. The consuming component renders a ConfirmDialog when `isBlocked` is true.

    2. Write tests in `web/src/hooks/useFormGuard.test.ts`:

       - Test that `proceed` and `reset` are forwarded from useBlocker.
       - Test that `enableBeforeUnload` is wired to isDirty.

    Note on pitfall: Per RESEARCH.md Pitfall 3, always use `reset(serverData)` when loading async data into forms to establish a clean baseline. This hook itself does not handle that -- consuming components must call `form.reset(data)` after query loads.
