- kind: IngressRoute

  - routes:
    - match: Host(`run.civpulse.org`)
    - kind: Rule
    - services:

      - port: 8000

  <name>Task 2: Create ArgoCD Application manifest</name>
  <files>k8s/argocd-app.yaml</files>
  <action>
Create `k8s/argocd-app.yaml`:
- apiVersion: argoproj.io/v1alpha1
- kind: Application
- metadata:
  - name: `run-api-dev`
  - namespace: `argocd` (ArgoCD Application resources live in the argocd namespace)
- spec:
  - project: `default` (matches existing voter-api-dev and contact-api apps)
  - source:
    - repoURL: `https://github.com/civicpulse/run-api.git`
    - targetRevision: `main`
    - path: `k8s`
  - destination:
    - server: `https://kubernetes.default.svc`
    - namespace: `run-api`
  - syncPolicy:
    - automated:
      - selfHeal: true
      - prune: true
    - syncOptions:
      - CreateNamespace=true

Add a comment at the top explaining:
- This file is stored in k8s/ for version control but is NOT auto-synced by ArgoCD
- It must be applied once manually: `kubectl apply -n argocd -f k8s/argocd-app.yaml`
- ArgoCD then watches the k8s/ directory for the app manifests (deployment, service, configmap, ingress)
- The argocd-app.yaml itself targets the `argocd` namespace so ArgoCD will not try to re-apply it to `run-api`
  </action>
