    Each section structure:
    ```
    Section header: "Phone Numbers" + [+ Add phone] button (RequireRole manager)
    ─────────────────────────────
    [☆/★] [value] [type badge] [✏ pencil] [🗑 trash]   ← contact row
    ↳ [inline edit form]                                ← expands below when pencil clicked
    ...
    [inline add form]                                   ← expands when + Add clicked
    ```

    Contact row:
    - Star: `<button onClick={() => setPrimary.mutate({ contactType: "phones", contactId: contact.id })}>`
      - Use `Star` icon (filled) if `contact.is_primary`, `Star` icon (outline/empty fill) if not

    - Value: phone_number / email / formatted address (line1, city state zip)
    - Type badge: `<Badge variant="secondary">{contact.phone_type}</Badge>`
    - Edit pencil: `<button onClick={() => setExpandedEdit(contact.id)}><Pencil className="size-4" /></button>` wrapped in RequireRole(manager)
    - Delete trash: `<button onClick={() => deletePhone.mutate(contact.id)}><Trash2 className="size-4" /></button>` wrapped in RequireRole(manager) + confirm with DestructiveConfirmDialog

    Inline edit form (appears below row when expandedEdit === contact.id):
    - Use react-hook-form with zod. For phone: `{ value: string, phone_type: string (select) }`. For email: `{ email: string, email_type: string }`. For address: `{ line1, line2, city, state, zip_code, address_type }`.
    - Save/Cancel buttons
    - On save: call useUpdatePhone/Email/Address.mutate, close form on success (setExpandedEdit(null)), toast.success
    - On cancel: setExpandedEdit(null)

    Inline add form (appears at bottom of section when showAddForm === "phones"):
    - Same fields as edit form but empty defaults
    - On save: call useCreatePhone/Email/Address.mutate, close form on success, toast.success

    Loading state: if phones/emails/addresses queries are loading, show Skeleton rows.
    Empty state: if no contacts, show EmptyState with appropriate message + "Add phone/email/address" button.
