<!-- Per CONTEXT.md — locked decisions for My Sessions -->
Route: /campaigns/$campaignId/phone-banking/my-sessions
Per-row columns: session name | status badge | call list | my check-in time | action button
Action button states:
  - session.status === "active" AND not yet checked in → [Check In] button
  - session.status === "active" AND already checked in (check_in_at set, no check_out_at) → [Resume Calling] link to /sessions/$sessionId/call
  - check_out_at is set → "Checked Out" (no action)
  - session.status !== "active" → "—" (no action available)

Note: PhoneBankSession does NOT include check-in data for the current user.
My Sessions page needs caller check-in state. Options:
  1. Fetch useSessionCallers for each session (N+1 problem — avoid)
  2. Use local state per row to track whether current user just checked in
  3. Show Check In button for all active sessions; rely on check-in mutation success to switch UI

Use option 3 (local state): Initialize a Set of checked-in session IDs from localStorage or useState([]).
After useCheckIn.mutate() succeeds for a session, add sessionId to the checked-in Set.
Show "Resume Calling" for sessions in the checked-in Set.

This is the simplest approach and works for the expected use case (caller loads the page, checks in, then starts calling immediately). The state resets on page refresh, which is acceptable for v1.

    ```typescript
    function RowAction({ session }: { session: PhoneBankSession }) {
      const { campaignId } = useParams(...)
      const checkIn = useCheckIn(campaignId, session.id)
      const isCheckedIn = checkedInSessionIds.has(session.id)

      if (session.status !== "active") return <span className="text-muted-foreground">—</span>
      if (isCheckedIn) {
        return (
          <Link to="/campaigns/$campaignId/phone-banking/sessions/$sessionId/call"
                params={{ campaignId, sessionId: session.id }}
                className="...button primary styles...">
            Resume Calling
          </Link>
        )
      }
      return (
        <Button
          size="sm"
          disabled={checkIn.isPending}
          onClick={() => checkIn.mutate(undefined, {
            onSuccess: () => {
              setCheckedInSessionIds(prev => new Set(prev).add(session.id))
              toast.success("Checked in successfully")
            },
            onError: () => toast.error("Check-in failed"),
          })}
        >
          {checkIn.isPending ? "Checking in..." : "Check In"}
        </Button>
      )
    }
    ```

    Note: useCheckIn is per-session (takes sessionId). The RowAction component calls useCheckIn with the session's specific ID. This means each row instantiates its own mutation. This is the correct pattern since mutations are cheap and per-row isolation is needed.
