     - Reason textarea (required, min 3 chars)

Create `web/src/routes/campaigns/$campaignId/volunteers/shifts/$shiftId/index.tsx`:

1. Route setup:
   - `createFileRoute("/campaigns/$campaignId/volunteers/shifts/$shiftId/")` with component ShiftDetailPage
   - Extract campaignId and shiftId from useParams

2. Data fetching:
   - `useShiftDetail(campaignId, shiftId)` for shift data (includes accurate signed_up_count and waitlist_count)
   - `useShiftVolunteers(campaignId, shiftId)` for roster data
   - `useVolunteerList(campaignId)` for volunteer name resolution

3. Volunteer name resolution (CRITICAL -- see RESEARCH.md Pitfall 1):
   - Build `volunteersById` lookup map using useMemo: `Record<string, { first_name: string; last_name: string }>`
   - Display name as `volunteersById[signup.volunteer_id]?.first_name + " " + volunteersById[signup.volunteer_id]?.last_name` with fallback to `signup.volunteer_id.slice(0, 8)` (ID substring)

4. Tab structure using shadcn Tabs component (same pattern as volunteer detail page):
   - Tab "Overview" (default)
   - Tab "Roster"

5. Overview Tab:
   - Shift name as heading with StatusBadge (shiftStatusVariant)
   - Info grid: Type (shiftTypeLabel badge), Date (formatted), Time range, Location (location_name + address if present, text only), Capacity bar: `{signed_up_count}/{max_volunteers} signed up` + if waitlist_count > 0: `({waitlist_count} waitlisted)`, Description (if present)
   - Edit button (RequireRole minimum="manager", only when shift.status === "scheduled"): opens ShiftDialog with editShift=shift
   - Contextual status transition buttons (RequireRole minimum="manager"):
     - Map VALID_TRANSITIONS[shift.status] to buttons:
     - "active" transition: [Activate] button (info/primary style)
     - "cancelled" transition: [Cancel Shift] button (destructive style, ConfirmDialog: "Are you sure you want to cancel this shift? All signups will be affected.")
     - "completed" transition: [Mark Complete] button (ConfirmDialog: "Mark this shift as complete?")
     - Use `useUpdateShiftStatus(campaignId, shiftId)` for all transitions
     - On success: toast("Shift status updated")
   - Self-signup/cancel section (below shift info, for non-managers or all users):
     - If shift.status === "scheduled": show Sign Up / Cancel Signup button
     - Use `useSelfSignup(campaignId, shiftId)` and `useCancelSignup(campaignId, shiftId)`
     - For Cancel: show ConfirmDialog "Cancel your signup for {shift.name}?"
     - Handle 422 "already signed up" with toast
     - Handle 404 "Not Registered" with toast "You need to register as a volunteer first"
     - Note: we cannot easily determine current user's signup status from the data. Show both Sign Up and Cancel buttons. The backend will return appropriate errors. Alternatively, scan the volunteers list for a match with current user -- but ShiftSignupResponse has volunteer_id not user_id. For v1, show Sign Up button; if it 422s, switch to showing Cancel.

6. Roster Tab:
   - DataTable with columns:
     - Name: resolved from volunteersById lookup
     - Status: StatusBadge with signupStatusVariant
     - Check In: formatted check_in_at datetime or "--"
     - Check Out: formatted check_out_at datetime or "--"
     - Hours: computed as `signup.adjusted_hours ?? (signup.check_out_at && signup.check_in_at ? ((new Date(signup.check_out_at).getTime() - new Date(signup.check_in_at).getTime()) / 3600000).toFixed(1) : "--")` (per RESEARCH.md Pitfall 8)
     - Actions: RowActions component (see below)

   - RowActions component (per-row, instantiates hooks per row -- per Phase 16/17 pattern):
     - Inline check-in/out buttons (RequireRole minimum="manager", only when shift.status === "active"):
       - signed_up status: [Check In] button -> useCheckInVolunteer, toast("Volunteer checked in"), single click no dialog
       - checked_in status: [Check Out] button -> useCheckOutVolunteer, toast("Volunteer checked out"), single click no dialog
       - checked_out status: shows computed hours (no action buttons)
       - waitlisted status: no action buttons
     - Kebab menu (RequireRole minimum="manager"):
       - "Remove Volunteer" -> useRemoveVolunteer with ConfirmDialog
       - "Adjust Hours" (only when status === "checked_out") -> opens AdjustHoursDialog with computedHours calculated from check_in_at/check_out_at

   - "+ Assign Volunteer" button above table (RequireRole minimum="manager"):
     - Opens AssignVolunteerDialog with existingVolunteerIds = array of all signup.volunteer_id values

   - Empty state: EmptyState title="No volunteers signed up", description based on role:
     - Manager: "No volunteers signed up" + [Assign Volunteer] CTA
     - Volunteer: "No volunteers signed up yet"

7. Loading state: Show loading spinner/skeleton while shift detail is loading. Show "Shift not found" if query errors.

8. Back navigation: Link back to shift list at top of page.
