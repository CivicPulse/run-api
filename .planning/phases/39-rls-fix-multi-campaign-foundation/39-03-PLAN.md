<objective>
Investigate campaign visibility root cause, fix multi-campaign membership creation, fix get_campaign_from_token scoping, and backfill existing data so users see all their org's campaigns.

Purpose: `ensure_user_synced()` currently uses `.limit(1)` to create a CampaignMember for only one campaign per org, breaking the campaign list for users in orgs with multiple campaigns. `get_campaign_from_token()` also uses `.limit(1)` and should be restricted to campaign list page fallback only. This fix removes the limits, adds a loop for all campaigns, and creates an Alembic data migration to backfill missing records in production. Per D-07, D-08, D-09, D-10, D-12.

Output: Root cause diagnosis, fixed `ensure_user_synced()`, fixed `get_campaign_from_token()`, Alembic data migration, and tests proving multi-campaign membership works.
</objective>

    ```python
    """Backfill missing CampaignMember records for multi-campaign orgs.

    Revision ID: 014_backfill_members
    Revises: {head_revision_from_013}
    """
    from alembic import op

    revision = "014_backfill_members"
    down_revision = "{get from alembic heads}"
    branch_labels = None
    depends_on = None


    def upgrade() -> None:
        # For each existing campaign_member, find the user's org via campaign ->
        # organization, then create membership for ALL campaigns in that org.
        # Uses ON CONFLICT to safely handle existing records (idempotent).
        op.execute("""
            INSERT INTO campaign_members (id, user_id, campaign_id, synced_at)
            SELECT
                gen_random_uuid(),
                cm_existing.user_id,
                c_other.id,
                NOW()
            FROM campaign_members cm_existing
            JOIN campaigns c_source ON cm_existing.campaign_id = c_source.id
            JOIN campaigns c_other ON c_other.organization_id = c_source.organization_id
            WHERE c_other.id != c_source.id
            ON CONFLICT (user_id, campaign_id) DO NOTHING
        """)


    def downgrade() -> None:
        # Data migration — downgrade is a no-op since we can't determine
        # which records were backfilled vs. originally created.
        pass
    ```

    **Key design decisions:**
    - Uses existing `campaign_members` as the bootstrap source (per RESEARCH.md recommendation) — for each user who has at least one membership, find their org via the campaign, then create memberships for all OTHER campaigns in that org.
    - `ON CONFLICT (user_id, campaign_id) DO NOTHING` uses the existing `uq_user_campaign` unique constraint for idempotency.
    - Downgrade is no-op — we cannot distinguish backfilled from original records.
