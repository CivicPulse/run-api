<objective>
Centralize RLS context setting into a single `get_campaign_db` dependency and remove all 244 inline `set_campaign_context` calls across 17 route files.

Purpose: The current pattern requires every endpoint handler to manually call `set_campaign_context(db, str(campaign_id))`. If a developer forgets this call, data leaks silently. By centralizing into a FastAPI dependency that extracts `campaign_id` from the URL path parameter, no endpoint can skip RLS context. Per D-04 and D-05.

Output: New `get_campaign_db` dependency in `app/api/deps.py`, all 17 route files refactored to use it, zero remaining inline `set_campaign_context` calls in route files.
</objective>

        campaign_id: uuid.UUID,
    ) -> AsyncGenerator[AsyncSession]:
        """DB session with RLS context auto-set from URL path parameter.

        All campaign-scoped routes MUST use this instead of get_db().
        campaign_id is extracted automatically by FastAPI from the URL path.
        Per D-04: centralizes RLS so no endpoint can skip it.
        Per D-05: only routes with campaign_id in path get RLS context.

        async with async_session_factory() as session:
            await set_campaign_context(session, str(campaign_id))
            yield session
