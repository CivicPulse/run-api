<objective>
Fix the critical RLS data isolation bug and add defense-in-depth pool checkout reset.

Purpose: The `set_config('app.current_campaign_id', :id, false)` call uses session-scoped config that persists across connection pool reuse, allowing campaign A's data to leak to campaign B. This is an active production security vulnerability. The fix changes the third parameter to `true` (transaction-scoped) so config auto-resets at COMMIT/ROLLBACK. A pool checkout event provides defense-in-depth.

Output: Fixed `app/db/rls.py`, pool checkout event in `app/db/session.py`, and test-first failing tests that prove the leak before fix and pass after.
</objective>
