Create `web/src/hooks/useCanvassingWizard.ts` -- the orchestrator hook that ties Zustand store state + React Query data into a single interface for the wizard route.

```typescript
import { useMemo, useCallback, useRef, useEffect } from "react"
import { useCanvassingStore } from "@/stores/canvassingStore"
import { useEnrichedEntries, useDoorKnockMutation, useSkipEntryMutation } from "@/hooks/useCanvassing"
import { groupByHousehold, SURVEY_TRIGGER_OUTCOMES, AUTO_ADVANCE_OUTCOMES } from "@/types/canvassing"
import type { Household, DoorKnockResultCode, EnrichedWalkListEntry } from "@/types/canvassing"
import { toast } from "sonner"
```

The hook signature:
```typescript
export function useCanvassingWizard(campaignId: string, walkListId: string)
```

Internal logic:
1. Call `useEnrichedEntries(campaignId, walkListId)` to get entries data
2. Read store state: `const { currentAddressIndex, completedEntries, skippedEntries, setWalkList, advanceAddress, jumpToAddress, skipEntry, touch } = useCanvassingStore()`
3. Call `useDoorKnockMutation(campaignId, walkListId)` and `useSkipEntryMutation(campaignId, walkListId)`

Derived state via `useMemo`:
- `households`: `groupByHousehold(entries ?? [])` -- the address-ordered list
- `currentHousehold`: `households[currentAddressIndex]` or null if out of bounds
- `totalAddresses`: `households.length`
- `completedAddresses`: count of households where ALL entries have a completedEntries[id] or skippedEntries includes id
- `isComplete`: `currentAddressIndex >= totalAddresses` (wizard finished)
- `activeEntryId`: for the current household, find the first entry whose id is NOT in completedEntries and NOT in skippedEntries. This is the voter the volunteer is currently interacting with. If all entries are completed/skipped, return null.

Key callbacks:

`handleOutcome(entryId: string, voterId: string, result: DoorKnockResultCode)`:
- Call doorKnockMutation.mutate with `{ walk_list_entry_id: entryId, result_code: result, voter_id: voterId }`
- Check if this is the FIRST "not_home" at a multi-voter household: if result === "not_home" AND household has >1 entries AND no other entries are completed yet, return `{ bulkPrompt: true, entryId, result }` (signal to UI to show bulk apply dialog)
- If `AUTO_ADVANCE_OUTCOMES.has(result)`:
  - After 300ms delay, check if all voters at current address are now handled (completed or skipped). If yes, call `advanceAddress()`. If not, the UI just moves to the next activeEntryId automatically.
- If `SURVEY_TRIGGER_OUTCOMES.has(result)`:
  - Return `{ surveyTrigger: true }` (signal to UI to open survey panel)
  - After survey completes/skips, the UI calls `handlePostSurveyAdvance()`

`handlePostSurveyAdvance()`:
- Check if all voters at current address are handled. If yes, call `advanceAddress()`.

`handleSkipAddress()`:
- For each entry in `currentHousehold.entries` that is not yet completed, call `skipEntry(entry.id)` in store and `skipEntryMutation.mutate(entry.id)`
- Then call `advanceAddress()` after 300ms

`handleBulkNotHome(entries: EnrichedWalkListEntry[])`:
- For each entry, sequentially call doorKnockMutation.mutate with result_code "not_home"
- After all mutations queued, wait 500ms for visual feedback, then call `advanceAddress()`

`handleJumpToAddress(index: number)`:
- Call `jumpToAddress(index)`

Return object:
```typescript
return {
  // Data
  households,
  currentHousehold,
  currentAddressIndex,
  totalAddresses,
  completedAddresses,
  activeEntryId,
  completedEntries,
  isComplete,
  isLoading: entriesQuery.isLoading,
  isError: entriesQuery.isError,

  // Actions
  handleOutcome,
  handlePostSurveyAdvance,
  handleSkipAddress,
  handleBulkNotHome,
  handleJumpToAddress,
}
```

Use `useEffect` to initialize the store: if `walkListId` differs from store's `walkListId`, call `setWalkList(walkListId)`.

Use `useEffect` to call `touch()` every 60 seconds while the wizard is active (for stale detection).

The `handleOutcomeWithBulk` wrapper:
- Calls `handleOutcome(entryId, voterId, result)`
- If result is "not_home" AND currentHousehold has >1 entries AND this is the first outcome at this address (no other entries completed):
  - Get remaining unhandled entries at this address
  - Show a sonner toast with custom content: "Apply to all {N} voters at this address?" with "Yes" and "No" buttons
  - "Yes" calls `handleBulkNotHome(remainingEntries)` and dismisses toast
  - "No" dismisses toast (normal flow continues, each voter gets individual outcome)
- Otherwise: normal flow (auto-advance handled by the hook)

Bulk "Not Home" toast implementation:
```typescript
const handleOutcomeWithBulk = useCallback((entryId: string, voterId: string, result: DoorKnockResultCode) => {
  const response = handleOutcome(entryId, voterId, result)

  // Check for bulk Not Home prompt
  if (result === "not_home" && currentHousehold && currentHousehold.entries.length > 1) {
    const remaining = currentHousehold.entries.filter(
      e => e.id !== entryId && !completedEntries[e.id] && !skippedEntries?.includes(e.id)
    )
    if (remaining.length > 0 && Object.keys(completedEntries).filter(id => currentHousehold.entries.some(e => e.id === id)).length === 0) {
      // This is the first outcome at a multi-voter household
      toast(`Apply to all ${remaining.length + 1} voters at this address?`, {
        action: {
          label: "Yes",
          onClick: () => handleBulkNotHome(remaining),
        },
        cancel: {
          label: "No",
          onClick: () => {},
        },
        duration: 10000,
      })
    }
  }
}, [handleOutcome, currentHousehold, completedEntries, handleBulkNotHome])
```
