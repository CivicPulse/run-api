## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, badge padding |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, outcome grid gap |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: Touch targets use 44px minimum (WCAG 2.5.5) for all interactive elements (outcome buttons, call buttons, skip button, end session button). Outcome grid gap is 12px (gap-3) matching canvassing pattern.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (text-base) | 400 (font-normal) | 1.5 |
| Label | 14px (text-sm) | 400 (font-normal) | 1.5 |
| Heading | 20px (text-xl) | 600 (font-semibold) | 1.2 |
| Caption | 12px (text-xs) | 400 (font-normal) | 1.5 |

Notes:
- Voter name: 16px / 400 (text-base font-normal) -- matches VoterCard pattern from Phase 31
- Phone number display: 16px / 400 (text-base font-normal) -- readable at glance
- Phone type label: 12px / 400 (text-xs) -- secondary context (Cell, Home, Work)
- Progress text: 14px / 400 (text-sm text-muted-foreground) -- matches FieldProgress
- Prior attempt text: 12px / 400 (text-xs text-muted-foreground) -- tertiary info
- Completion summary heading: 20px / 600 -- celebratory emphasis

## Interaction Contracts

### Start Session
- Volunteer taps "Start Calling" on landing hub assignment card
- System auto-checks in to active session, claims first batch of 5 entries
- First voter card renders with phone numbers and outcome grid
- FieldProgress shows "0 of {total} calls"

### Call Flow
- Volunteer taps "Call" button next to a phone number
- Native dialer opens via `tel:+15551234567` link
- System records `callStartedAt` timestamp and `phoneNumberUsed` in Zustand store
- After call, volunteer returns to app and taps outcome button
- System records call with `call_started_at`, `call_ended_at` (now), `phone_number_used`
- If outcome is "Answered": InlineSurvey sheet opens from bottom
- If outcome is anything else: auto-advance to next voter

### Skip
- Volunteer taps "Skip" button
- Entry released back to AVAILABLE via `self_release_entry()`
- Auto-advance to next voter

### Long-Press Copy
- Volunteer long-presses any phone number text
- E.164 number copied to clipboard via `navigator.clipboard.writeText()`
- Sonner toast shows "Copied"

### Pre-fetch
- When 2 entries remain in current batch, claim next batch of 5 in background
- Append new entries to Zustand store via `addEntries()`
- No loading spinner visible to volunteer

### End Session (explicit)
- Volunteer taps "End Session" button
- AlertDialog confirmation: "Done calling?" / "Your progress is saved. You can resume later."
- Confirm: check-out + release uncalled entries, navigate to hub
- Cancel: dismiss dialog, continue calling

### End Session (back arrow)
- Same AlertDialog confirmation as explicit end session
- Confirm: check-out + release + navigate to hub

### Session Complete (all entries done)
- CompletionSummary renders with session stats
- "Back to Hub" button navigates to field landing

### ARIA & Accessibility (A11Y-05)
- `aria-live="polite"` region announces voter name on each transition: "Now calling {Name}"
- Outcome buttons have `aria-label` with full text (e.g., "Record outcome: Answered")
- Phone "Call" buttons have `aria-label`: "Call {Name} at {display number}"
- Skip button has `aria-label`: "Skip this voter and move to next"
- FieldProgress has `role="status"` with `aria-label`: "{N} of {M} calls completed"
- Terminal phone numbers have `aria-label`: "{display number} -- wrong number" or "disconnected"
