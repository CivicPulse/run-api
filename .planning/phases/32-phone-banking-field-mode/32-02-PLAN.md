**A. Create `web/src/hooks/useCallingSession.ts`** -- orchestrator hook paralleling useCanvassingWizard:

```typescript
import { useMemo, useCallback, useEffect, useRef } from "react"
import { useCallingStore } from "@/stores/callingStore"
import {
  useCheckIn,
  useCheckOut,
  useClaimEntry,
  useRecordCall,
  useSelfReleaseEntry,
  usePhoneBankSession,
} from "@/hooks/usePhoneBankSessions"
import { useCallListDetail } from "@/hooks/useCallLists"  // need to verify this hook exists, if not use inline query
import { CALL_SURVEY_TRIGGER } from "@/types/calling"
import type { SessionStats } from "@/types/calling"
```

The hook signature: `useCallingSession(campaignId: string, sessionId: string)`

Key behaviors:
1. **Initialization flow**: On mount, fetch session detail via `usePhoneBankSession(campaignId, sessionId)` to get `call_list_id`. Then auto check-in via `useCheckIn`. Then claim first batch of 5 entries via `useClaimEntry(campaignId, callListId)` with `batch_size: 5`. Initialize store via `startSession()`.
   - NOTE: `useClaimEntry` currently hardcodes `batch_size: 1`. The hook must override this -- either modify useClaimEntry to accept batch_size param, or create an inline mutation using `api.post(..., { json: { batch_size: 5 } })`.
   - Use a `useRef(false)` guard to prevent double-initialization in StrictMode.

2. **Pre-fetch**: When `remainingEntries <= 2`, trigger claim of next 5 entries in background. Append via `addEntries()`. Use `useRef` to prevent duplicate pre-fetch.

3. **Derived state**:
   - `currentEntry`: `entries[currentEntryIndex]` or null
   - `completedCount`: `Object.keys(completedCalls).length`
   - `totalEntries`: read from `FieldMeResponse.phone_banking.total` (passed as prop) or `entries.length` as fallback
   - `remainingEntries`: `entries.length - currentEntryIndex - Object.keys(completedCalls).length` (approximately)
   - `isComplete`: `currentEntryIndex >= entries.length` AND no more entries to claim (claim returns empty)
   - `sessionStats`: computed from completedCalls values

4. **handleOutcome(resultCode: string)**:
   - Get entryId from `currentEntry.id`
   - Get `callStartedAt` and `phoneNumberUsed` from store. If `callStartedAt` is null, use current ISO timestamp for both start and end (volunteer recorded outcome without tapping Call first).
   - Call `recordCall.mutate({ call_list_entry_id: entryId, result_code: resultCode, phone_number_used: phoneNumberUsed || "", call_started_at: callStartedAt || now, call_ended_at: now })`
   - Call `store.recordOutcome(entryId, resultCode)`
   - If `resultCode === CALL_SURVEY_TRIGGER` ("answered"): return `{ surveyTrigger: true }`
   - Otherwise: auto-advance after 300ms via `setTimeout(() => advanceEntry(), 300)`
   - Return `{ surveyTrigger: false }`

5. **handlePostSurveyAdvance()**: Call `store.advanceEntry()`

6. **handleSkip()**:
   - Get entryId from `currentEntry.id`
   - Call `selfRelease.mutate(entryId)`
   - Call `store.skipEntry(entryId)`
   - Advance after 300ms

7. **handleEndSession()**:
   - Call `checkOut.mutate()`
   - Call `store.reset()`
   - Return -- caller navigates to hub

8. **handleCallStarted(phone: string)**: Call `store.setCallStarted(phone)`

9. **Touch timer**: `setInterval(() => touch(), 60_000)` like canvassing

10. **scriptId**: Fetch call list detail to get `script_id`. Use `usePhoneBankSession` data which has `call_list_id`, then either fetch call list detail or embed script_id. Check if `useCallListDetail` hook exists. If not, create an inline useQuery or use the call-lists endpoint: `GET /api/v1/campaigns/{campaignId}/call-lists/{callListId}`.

Return object:
```typescript
{
  currentEntry, completedCount, totalEntries, isComplete, sessionStats,
  isLoading, isError, error, scriptId,
  handleOutcome, handlePostSurveyAdvance, handleSkip, handleEndSession, handleCallStarted,
  noEntriesAvailable,  // true if claim returned empty on first try
}
```
