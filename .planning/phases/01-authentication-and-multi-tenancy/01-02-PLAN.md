    1. **Create `app/services/zitadel.py`:**
       - `ZitadelService` class with httpx.AsyncClient
       - `__init__(settings)`: Store issuer URL, client ID, client secret from settings
       - `_get_token()`: POST to `/oauth/v2/token` with grant_type=client_credentials, scope includes `urn:zitadel:iam:org:project:id:zitadel:aud`. Cache token, refresh on expiry or 401.
       - `create_organization(name: str) -> dict`: POST to `/management/v1/orgs` with Bearer token. Returns org dict with `id` field. Raises `ZitadelUnavailableError` on connection error or 5xx.
       - `deactivate_organization(org_id: str) -> None`: POST to deactivate endpoint. Used for soft-delete.
       - `delete_organization(org_id: str) -> None`: DELETE org. Used only for compensating transaction cleanup.
       - `assign_project_role(org_id: str, user_id: str, role: str) -> None`: Assign a project role to a user within an org.
       - `remove_project_role(org_id: str, user_id: str, role: str) -> None`: Remove role grant.
       - Use `httpx.AsyncClient` as a context manager within each method (not stored as instance attribute) to avoid lifecycle issues.

    2. **Create `app/services/campaign.py`:**
       - `CampaignService` class (or module-level functions, per preference).
       - `create_campaign(db, data, user, zitadel)`: Compensating transaction pattern per RESEARCH.md Pattern 4. Create ZITADEL org first. If local DB fails, delete ZITADEL org. Create campaign_member record for the creator. Return campaign.
       - `get_campaign(db, campaign_id)`: Simple select by ID.
       - `list_campaigns(db, cursor, limit)`: Cursor-based pagination. Use campaign.created_at + campaign.id for cursor stability. Return items + pagination metadata.
       - `update_campaign(db, campaign_id, data)`: Update allowed fields. Only admin+ can call this (enforced at route level).
       - `delete_campaign(db, campaign_id, user, zitadel)`: Verify user is owner (created_by). Set status to "deleted", set updated_at. Call zitadel.deactivate_organization. Fail-fast if ZITADEL is down (503).

       - Test all CRUD operations
       - Follow RED -> GREEN -> REFACTOR

    1. **Create Pydantic schemas:**
       - `app/schemas/campaign.py`:
         - `CampaignCreate`: name (str, min 3 max 100), type (CampaignType enum), jurisdiction_fips (str|None), jurisdiction_name (str|None), election_date (date|None), candidate_name (str|None), party_affiliation (str|None).
         - `CampaignUpdate`: All fields optional (name, type, jurisdiction_fips, jurisdiction_name, election_date, candidate_name, party_affiliation, status). Only allow status transitions: active->suspended, active->archived, suspended->active.
         - `CampaignResponse`: All campaign fields + id, created_at, updated_at. Extends BaseSchema (from_attributes=True).
       - `app/schemas/user.py`:
         - `UserResponse`: id, display_name, email, created_at.
         - `UserCampaignResponse`: campaign_id, campaign_name, role (str).

    2. **Create `app/api/deps.py`:**
       - `get_db_with_rls(campaign_id)`: Dependency that gets a DB session AND sets the RLS campaign context via `set_campaign_context()`. Used by all campaign-scoped endpoints.
       - `get_campaign_from_token(user: AuthenticatedUser, db: AsyncSession)`: Looks up campaign by user.org_id (matches campaigns.zitadel_org_id). Returns campaign or raises 404.
       - `ensure_user_synced(user: AuthenticatedUser, db: AsyncSession)`: Creates or updates local User record from JWT claims on each request. Creates CampaignMember record if not exists (belt-and-suspenders per CONTEXT.md). This is a dependency that runs on all authenticated endpoints.

    3. **Create `app/api/v1/campaigns.py`:**
       - `POST /api/v1/campaigns`: Accepts CampaignCreate. Any authenticated user can create (they become owner). Uses CampaignService.create_campaign. Returns 201 + CampaignResponse.
       - `GET /api/v1/campaigns`: List campaigns for current user's org context. Uses cursor pagination (limit param, cursor param). Returns `{items: [...], pagination: {next_cursor, has_more}}`. Requires viewer+ via require_role("viewer").
       - `GET /api/v1/campaigns/{campaign_id}`: Get single campaign. Requires viewer+. RLS ensures you can only see your own campaign.
       - `PATCH /api/v1/campaigns/{campaign_id}`: Partial update. Requires admin+ via require_role("admin"). Returns CampaignResponse.
       - `DELETE /api/v1/campaigns/{campaign_id}`: Soft-delete. Requires owner via require_role("owner"). Returns 204.
       - All error responses use RFC 9457 format (handled by fastapi-problem-details).

    4. **Create `app/api/v1/users.py`:**
       - `GET /api/v1/me`: Returns current user from local DB (synced from JWT). Returns UserResponse.
       - `GET /api/v1/me/campaigns`: Returns all campaigns the user belongs to. Query campaign_members join campaigns. Returns list of UserCampaignResponse.

    5. **Wire routers in `app/api/v1/router.py`:**
