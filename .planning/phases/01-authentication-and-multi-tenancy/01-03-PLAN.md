  <action>
    1. **Create `app/models/invite.py`:**
       - `Invite` model: id (UUID PK), campaign_id (UUID FK campaigns.id), email (str, indexed), role (str -- the role being granted: "admin", "manager", etc.), token (UUID, unique, indexed, default uuid4), expires_at (datetime), accepted_at (datetime|None), revoked_at (datetime|None), created_by (str FK users.id), created_at (datetime).
       - Add Invite import to `app/db/base.py` so Alembic detects it.

    2. **Create Alembic migration `002_invites_table.py`:**
       - Creates invites table.
       - Enables RLS on invites table.
       - Creates RLS policy: `USING (campaign_id = current_setting('app.current_campaign_id', true)::uuid)`.
       - Grants permissions to app_user.

    3. **Create `app/schemas/invite.py`:**
       - `InviteCreate`: email (EmailStr), role (str, must be valid CampaignRole name, cannot be "owner").
       - `InviteResponse`: id, campaign_id, email, role, token (only in create response), expires_at, accepted_at, revoked_at, created_at.
       - `InviteAcceptResponse`: message, campaign_id, role.

    4. **Create `app/services/invite.py`:**
       - `create_invite(db, campaign_id, data, creator, zitadel)`:
         - Verify creator can invite at this role level: owners can invite any role except owner. Admins can invite manager and below. Others cannot invite.
         - Check no pending invite exists for same email + campaign.
         - Create Invite record with 7-day expiry, UUID v4 token.
         - ZITADEL sends the invite email using its built-in templates (per CONTEXT.md). The API uses ZITADEL's CreateInviteCode to trigger the email if the user doesn't exist in ZITADEL yet, or just stores the invite if they do.
         - Return the invite (include token in response for URL construction).
       - `validate_invite(db, token)`: Look up by token. Check not expired, not revoked, not accepted. Return invite or None.
       - `accept_invite(db, token, user, zitadel)`:
         - Validate invite. Verify user.email matches invite.email (case-insensitive).
         - Create or update CampaignMember record.
         - Call zitadel.assign_project_role(org_id, user.id, invite.role).
         - Set invite.accepted_at = now().
         - Return success.
       - `revoke_invite(db, invite_id)`: Set revoked_at. Return success.
       - `list_invites(db, campaign_id)`: Return pending (not accepted, not revoked, not expired) invites.

    5. **Create `app/api/v1/invites.py`:**
       - `POST /api/v1/campaigns/{campaign_id}/invites`: Create invite. Requires admin+ (via require_role("admin")). Returns 201 + InviteResponse with token.
       - `GET /api/v1/campaigns/{campaign_id}/invites`: List pending invites. Requires admin+. Returns list.
       - `DELETE /api/v1/campaigns/{campaign_id}/invites/{invite_id}`: Revoke. Requires admin+. Returns 204.
       - `POST /api/v1/invites/{token}/accept`: Accept invite. Requires authentication (any role -- the user is joining). Returns InviteAcceptResponse.

    6. **Wire invites router in `app/api/v1/router.py`.**

    7. **Write tests:**
       - `tests/unit/test_invite_service.py`: Test all service methods with mocked DB and ZITADEL.
       - `tests/unit/test_api_invites.py`: Test API endpoints with mocked dependencies.
       - Follow RED -> GREEN -> REFACTOR.

       - `MemberResponse`: user_id, display_name, email, role (str), synced_at.
       - `RoleUpdate`: role (str, valid CampaignRole name, cannot be "owner" unless transferring).
       - `OwnershipTransfer`: new_owner_id (str).

    2. **Create `app/api/v1/members.py`:**
       - `GET /api/v1/campaigns/{campaign_id}/members`: List members. Requires viewer+. Query campaign_members joined with users. For each member, determine their role -- since roles are in ZITADEL JWT, for listing purposes call ZITADEL to get project role grants for the org, or use a cached approach.        - `POST /api/v1/campaigns/{campaign_id}/transfer-ownership`: Transfer ownership. Requires owner. Accepts OwnershipTransfer body. Demotes current owner to admin, promotes target to owner. Updates ZITADEL roles and campaign.created_by. Returns 200.
