1. Update persist config:
```ts
persist((set) => ({...}), {
  name: "offline-queue",
  version: 1,
  storage: createJSONStorage(() => localStorage),
  partialize: (state) => ({ items: state.items }),
  migrate: (persistedState, version) => {
    if (version === 0) {
      // v0 → v1: stamp missing client_uuid from item.id
      const s = persistedState as { items?: QueueItem[] }
      if (Array.isArray(s?.items)) {
        s.items = s.items
          .filter((it) => it && typeof it.id === "string")
          .map((it) => ({
            ...it,
            payload: {
              ...it.payload,
              client_uuid: (it.payload as any).client_uuid ?? it.id,
            },
          }))
      }
      return s
    }
    return persistedState
  },
  onRehydrateStorage: () => (_state, error) => {
    if (error) console.warn("[offlineQueue] rehydrate error", error)
  },
})
```

2. Wrap `push()`'s set() in a try/catch. On QuotaExceededError, dynamic-import `sonner` and fire `toast.error`. Do NOT crash — leave the in-memory items array as-is (best-effort).

3. Write `offlineQueueStore.persistence.test.ts` covering behaviors 1-5. Use `vi.stubGlobal("localStorage", ...)` to inject controlled backends per test.
