<objective>
The current retry is "try every item on every tick" â€” no backoff, no budget, no
dead-letter. Audit Â§3 flagged this as OFFLINE-03's primary gap. This plan
replaces the flat retry with a classified-error strategy:

- **Network error / 5xx:** exponential backoff, keep retrying forever (subject
  to MAX_RETRY which we now raise much higher because the backoff itself
  throttles attempts).
- **4xx non-409 (validation, auth, forbidden):** never succeeds, move to
  dead-letter immediately for volunteer review.
- **409:** existing "success" path (server says already-have-it â†’ remove
  silently).
- **Budget:** 30s deadline from sync start; after that, mark the store
  `isSlow=true` so the pill UI can show "Syncing (slow)". Sync itself keeps
  going â€” budget is user-facing, not a hard cutoff.
</objective>
