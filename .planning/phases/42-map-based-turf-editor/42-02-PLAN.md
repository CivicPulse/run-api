**3. Create `web/src/components/canvassing/map/GeomanControl.tsx`:**

Per research Pattern 1 — wrap Geoman as a react-leaflet control using `createControlComponent`.

```typescript
import { createControlComponent } from "@react-leaflet/core"
import * as L from "leaflet"
import "@geoman-io/leaflet-geoman-free"
import "@geoman-io/leaflet-geoman-free/dist/leaflet-geoman.css"

interface GeomanOptions extends L.ControlOptions {
  position: L.ControlPosition
  drawPolygon?: boolean
  editMode?: boolean
  dragMode?: boolean
  removalMode?: boolean
}

const Geoman = L.Control.extend({
  options: {} as GeomanOptions,
  initialize(options: GeomanOptions) {
    L.setOptions(this, options)
  },
  addTo(map: L.Map) {
    if (!map.pm) return this
    map.pm.addControls({
      ...this.options,
      // Only polygon drawing and editing — disable everything else
      drawMarker: false,
      drawCircle: false,
      drawPolyline: false,
      drawRectangle: false,
      drawCircleMarker: false,
      drawText: false,
      cutPolygon: false,
      rotateMode: false,
    })
    return this
  },
})

export const GeomanControl = createControlComponent(
  (props: GeomanOptions) => new Geoman(props)
)
```

**1. Create `web/src/components/canvassing/map/TurfMapEditor.tsx`:**

Per D-01 (draw-then-confirm), D-02 (replaces textarea), D-13 (editor with conditional Geoman), D-17 (responsive height).

The component receives `value` (GeoJSON string from form) and `onChange` (callback to update form). It:
- Renders the MapProvider with GeomanControl
- On `pm:create` event, extracts `layer.toGeoJSON().geometry` and calls `onChange(JSON.stringify(geometry, null, 2))`
- On `pm:edit` event, extracts updated geometry and calls `onChange`
- When `value` changes externally (from JSON textarea), clears existing layers and renders the new GeoJSON
- If `defaultBoundary` prop is provided (edit mode), renders the existing boundary as an editable layer on mount
- Uses `useMap()` hook to fit bounds to the drawn/existing polygon
- Proper cleanup: `map.off("pm:create")` and `map.off("pm:edit")` in useEffect cleanup

```typescript
interface TurfMapEditorProps {
  value: string           // GeoJSON string from form field
  onChange: (value: string) => void  // Update form field
  defaultBoundary?: Record<string, unknown>  // Existing boundary for edit mode
}
```

Key implementation details:
- Use a `useRef` for the editable feature group layer
- Listen for `pm:create` — when user draws, remove any previous polygon first (only one polygon allowed), then call onChange
- Listen for `pm:edit` and `pm:remove` — update/clear the form value
- Use a `useEffect` watching `value` to handle JSON textarea -> map sync. Parse the JSON, validate it looks like a Polygon, clear layers, add new GeoJSON layer. Wrap in try/catch for invalid JSON (just ignore)
- Use a `syncFromJsonRef` boolean ref to prevent circular updates (map -> form -> map infinite loop). Set it true before calling onChange from map events, check it in the value-watching useEffect
- On mount with `defaultBoundary`, render the boundary and fit map bounds to it

**Inner component pattern** (must be inside MapContainer to use `useMap()`):
```typescript
function MapEditor({ value, onChange, defaultBoundary }: TurfMapEditorProps) {
  const map = useMap()
  // ... all the logic here
  return <GeomanControl position="topleft" drawPolygon editMode dragMode removalMode />
}

export function TurfMapEditor(props: TurfMapEditorProps) {
  return (
    <MapProvider>
      <MapEditor {...props} />
    </MapProvider>
  )
}
```
