**Phase 1: Run the full suite and collect all failures**

Run the entire E2E suite against local Docker Compose:
```bash
cd web && npx playwright test --reporter=line 2>&1
```

Capture all failures. If the output is too long, also run:
```bash
cd web && npx playwright test --reporter=json 2>&1 | jq '.suites[].specs[] | select(.ok == false) | {file: .file, title: .title, status: .tests[0].status}'
```

**Phase 2: Classify each failure**

For EVERY failing test, determine the root cause:

a) **Test bug** (spec logic error): Wrong locator, wrong URL, timing issue, missing wait, incorrect assertion text, stale test data assumption.
   - FIX immediately in the spec file.

b) **App bug** (real application defect): Feature not working, API returning errors, UI component broken, missing functionality.
   - Log in 60-BUGS.md with structured format.
   - Add `test.skip('BUG-XX: description')` to the failing test per D-08.

c) **Infrastructure issue**: Docker not running, auth expired, seed data missing.
   - Resolve the infrastructure issue (re-run seed, restart containers, etc.) and re-run.

**Phase 3: Fix all test bugs**

For each test bug identified:
1. Read the failing spec file
2. Read the app component/route source to understand expected behavior
3. Fix the spec (correct locator, add wait, fix assertion, update timing)
4. Re-run just that spec to verify the fix: `cd web && npx playwright test <spec-name> --reporter=line`

Common test bug fixes:
- Wrong `getByText` pattern -> read component source for actual rendered text
- Missing `waitForURL` or `waitForResponse` -> add appropriate wait
- Timeout too short -> increase timeout (use 15_000 for navigation, 30_000 for data loads)
- Locator matches multiple elements -> add `.first()` or more specific locator
- API helper URL wrong -> verify against actual API routes
