Key design decisions (per user locked decisions):
- Expansion covers General and Primary ONLY (no Runoff/Special -- L2 data doesn't have these)
- voted_in year-only uses .overlap() for OR semantics (EITHER election)
- not_voted_in year-only uses two separate ~.contains() for AND NOT semantics (skipped BOTH)
- Do NOT use ~.overlap() for not_voted_in -- it has NULL handling issues (see research pitfall 3)
- Detection via regex `^\d{4}$` -- simple, clear, handles edge cases
