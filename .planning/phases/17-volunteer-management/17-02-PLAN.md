    NOTE: The existing `useFieldOps.useVolunteers` hook is preserved for backward compatibility. The new hooks in `useVolunteers.ts` are separate and support filters.

    1. web/src/hooks/useVolunteers.ts:
       - `volunteerKeys` query key factory: `{ all: (cid) => ["volunteers", cid], detail: (cid, vid) => ["volunteers", cid, vid], list: (cid, filters) => ["volunteers", cid, "list", filters] }`
       - `useVolunteerList(campaignId, filters?: { status?: string; skills?: string; name?: string })`: useQuery with queryKey volunteerKeys.list(campaignId, filters). queryFn builds URLSearchParams from non-undefined filters (skills passed as comma-separated string), calls `api.get(\`api/v1/campaigns/${campaignId}/volunteers${qs ? "?" + qs : ""}\`).json<PaginatedResponse<VolunteerResponse>>()`. enabled: !!campaignId. Import PaginatedResponse from @/types/common.
       - `useVolunteerDetail(campaignId, volunteerId)`: useQuery with queryKey volunteerKeys.detail(campaignId, volunteerId). Fetches `api.get(\`api/v1/campaigns/${campaignId}/volunteers/${volunteerId}\`).json<VolunteerDetailResponse>()`. enabled: !!campaignId && !!volunteerId.
       - `useCreateVolunteer(campaignId)`: useMutation, mutationFn takes VolunteerCreate, POSTs to `/volunteers` with json body. onSuccess invalidates volunteerKeys.all(campaignId).
       - `useUpdateVolunteer(campaignId, volunteerId)`: useMutation, mutationFn takes VolunteerUpdate, PATCHes to `/volunteers/${volunteerId}`. onSuccess invalidates volunteerKeys.all(campaignId) AND volunteerKeys.detail(campaignId, volunteerId).
       - `useUpdateVolunteerStatus(campaignId, volunteerId)`: useMutation, mutationFn takes string status, PATCHes to `/volunteers/${volunteerId}/status` with `{ json: { status } }`. onSuccess invalidates both list and detail.
       - `useSelfRegister(campaignId)`: useMutation, mutationFn takes VolunteerCreate, POSTs to `/volunteers/register`. Does NOT auto-invalidate (caller handles redirect). The mutation should NOT use `.json()` on success to allow the caller to handle both 201 (success) and 409 (conflict with volunteer_id in body). Use `api.post(...).then(res => res.json())` for 201, and for error handling the caller should catch and parse the 409 response body to extract volunteer_id.

    2. web/src/hooks/useVolunteerTags.ts:
       - `volunteerTagKeys` factory: `{ all: (cid) => ["volunteer-tags", cid] }`
       - `useVolunteerCampaignTags(campaignId)`: useQuery fetching `api/v1/campaigns/${campaignId}/volunteer-tags` returning `VolunteerTagResponse[]`.
       - `useCreateVolunteerTag(campaignId)`: useMutation, POST with `{ json: { name } }`, invalidates volunteerTagKeys.all(campaignId).
       - `useUpdateVolunteerTag(campaignId, tagId)`: useMutation, PATCH `volunteer-tags/${tagId}` with `{ json: { name } }`, invalidates volunteerTagKeys.all(campaignId).
       - `useDeleteVolunteerTag(campaignId, tagId)`: useMutation, DELETE `volunteer-tags/${tagId}`, invalidates volunteerTagKeys.all(campaignId).
       - `useAddTagToVolunteer(campaignId, volunteerId)`: useMutation, POST `volunteers/${volunteerId}/tags/${tagId}` (tagId is the mutationFn arg). onSuccess invalidates volunteerKeys.detail(campaignId, volunteerId). Import volunteerKeys from useVolunteers.ts.
       - `useRemoveTagFromVolunteer(campaignId, volunteerId)`: useMutation, DELETE `volunteers/${volunteerId}/tags/${tagId}`. onSuccess invalidates volunteerKeys.detail(campaignId, volunteerId).

    3. web/src/hooks/useVolunteerAvailability.ts:
       - `useVolunteerAvailability(campaignId, volunteerId)`: useQuery fetching `volunteers/${volunteerId}/availability` returning `AvailabilityResponse[]`.
       - `useAddAvailability(campaignId, volunteerId)`: useMutation, POST with `{ json: { start_at, end_at } }`. onSuccess invalidates availability query AND volunteerKeys.detail (detail response includes availability).
       - `useDeleteAvailability(campaignId, volunteerId)`: useMutation, DELETE `volunteers/${volunteerId}/availability/${availabilityId}` (availabilityId is mutationFn arg). onSuccess invalidates same.

    4. web/src/hooks/useVolunteerHours.ts:
       - `useVolunteerHours(campaignId, volunteerId)`: useQuery fetching `volunteers/${volunteerId}/hours` returning `VolunteerHoursResponse`.
