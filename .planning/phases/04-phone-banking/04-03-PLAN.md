    **PhoneBankService** (`app/services/phone_bank.py`):

    Compose VoterInteractionService, SurveyService, DNCService, and CallListService (same composition pattern as CanvassService).

    ```python
    class PhoneBankService:
        def __init__(self):
            self._interaction_service = VoterInteractionService()
            self._survey_service = SurveyService()
            self._dnc_service = DNCService()
    ```

    **Session CRUD:**
    1. `create_session(session, campaign_id, data: PhoneBankSessionCreate, user_id)` -> PhoneBankSession
    2. `get_session(session, session_id)` -> PhoneBankSession
    3. `list_sessions(session, campaign_id)` -> list[PhoneBankSession]
    4. `update_session(session, session_id, data: PhoneBankSessionUpdate)` -> PhoneBankSession: Enforce status transitions: draft->{active}, active->{paused, completed}, paused->{active, completed}. On completion, force-release all IN_PROGRESS entries for that session's call_list (where claimed entries were claimed by callers assigned to this session).

    **Caller management:**
    5. `assign_caller(session, session_id, user_id)` -> SessionCaller
    6. `remove_caller(session, session_id, user_id)` -> None: Release caller's claimed entries first, then delete SessionCaller.
    7. `check_in(session, session_id, user_id)` -> SessionCaller: Set check_in_at = now. Session must be ACTIVE.
    8. `check_out(session, session_id, user_id)` -> SessionCaller: Set check_out_at = now. Release caller's claimed entries back to AVAILABLE.

    **Call recording:**
    9. `record_call(session, campaign_id, session_id, data: CallRecordCreate, user_id)` -> CallRecordResponse:
       - Load entry by data.call_list_entry_id, verify it's claimed by this user
       - Record PHONE_CALL interaction via _interaction_service with payload: {result_code, call_list_id (from entry), session_id, phone_number_used, call_started_at, call_ended_at, notes, survey_complete}
       - Update entry based on result_code:
         - ANSWERED: status=COMPLETED, increment attempt_count, set last_attempt_at
         - REFUSED/DECEASED: status=TERMINAL (person-level terminal)
         - WRONG_NUMBER/DISCONNECTED: add to phone_attempts JSONB for that phone. If other untried phones remain -> status=AVAILABLE (caller can try next number). If no phones left -> status=TERMINAL.
         - NO_ANSWER/BUSY/VOICEMAIL: increment attempt_count, set last_attempt_at, set status=AVAILABLE (will be recycled after cooldown). If attempt_count >= max_attempts -> status=MAX_ATTEMPTS.
       - If REFUSED: call _dnc_service.add_entry(campaign_id, phone_number_used, DNCReason.REFUSED, user_id)
       - If result_code == ANSWERED and data.survey_responses not empty: call _survey_service.record_responses(session, script_id, voter_id, campaign_id, data.survey_responses, user_id)
       - Update CallList.completed_entries count
       - Return CallRecordResponse with interaction_id

    **Supervisor operations:**
    10. `get_progress(session, session_id)` -> SessionProgressResponse: Query entry stats (COUNT by status) for the session's call_list. Query per-caller stats (COUNT calls, check_in/out) for session's callers.
    11. `reassign_entry(session, entry_id, new_caller_id)` -> CallListEntry: Update claimed_by.
    12. `force_release_entry(session, entry_id)` -> CallListEntry: Set status=AVAILABLE, clear claimed_by/claimed_at.
    13. `end_caller_session(session, session_id, user_id)` -> None: Check out caller + release their entries.

    Replace test stubs in test_phone_bank.py with real test implementations. Mock AsyncSession and composed services.
