    Loop over every row in `106-BASELINE.md §Pytest`. For each failing test, apply this exact procedure:

    **A. Pre-fix probe (< 1 min):**
    Re-run the single test to confirm current failure mode:
    ```bash
    uv run pytest <file>::<test_name> -x --tb=short
    ```
    If the test now passes (rare — environmental flake), note it in the final commit body and move on.

    **B. Categorization (< 1 min):**
    Consult pitfall 5 patterns. If the test file name matches `test_*canvassing*`, `test_*walk_list*`, `test_*active_state*`, `test_*leaflet*`, `test_*map_marker*`, `test_*offline*`, `test_*sync_queue*` — DELETE with a body line referencing the downstream phase (107/108/109/110). Do not spend fix time on it.

    **C. 15-min fix attempt (D-01):**
    Start a timer. If the fix is mechanical — outdated fixture, stale assertion, wrong import path, missing await, schema column rename — fix it in the test file only. Avoid touching `tests/conftest.py`, `tests/unit/conftest.py`, `tests/integration/conftest.py` (Pitfall 3). If a shared fixture change is truly unavoidable, note it in the commit body and run the FULL pytest suite after the change (`uv run pytest`) to catch ripple breakage.

    If the test fails because of a real product bug (D-08) — DO NOT fix the product code. Delete the test with a justification referencing the bug, and record the bug as a pending todo:
    ```bash
    mkdir -p .planning/todos/pending
    cat > .planning/todos/pending/pytest-<short-slug>.md <<'EOF'
    # Pending bug: <one-line description>

    Source: deleted test tests/<path>::<test_name> in phase 106.
    Symptom: <what the test was asserting and why it fails>.
    Owner phase: <107/108/109/110 or unassigned>.
    EOF
    ```

    If the time box is exceeded → DELETE with justification.

    **D. Per-batch commit (D-14):**
    After each batch of ~3-10 related tests (or a single file's worth, whichever is smaller), commit. Group fixes and deletions into the same commit when they touch the same module. Use this exact commit template:

    ```bash
    uv run ruff check tests/<path>
    uv run ruff format tests/<path>
    git add tests/<touched_files>
    # If deletions in this batch:
    git rm tests/<deleted_files>
    git commit -m "test(pytest): triage <module> — N fixed, M deleted

    Fixes:
    - tests/path/test_foo.py::test_bar — <one-line what was wrong and the fix>
    - tests/path/test_foo.py::test_baz — <one-line>

    PHASE-106-DELETE:
    - tests/path/test_removed.py — <one-line justification>
      reason: <covers removed feature X | redesign exceeded 15min box | product bug #Y deferred per D-08 | imminent rewrite in phase 107 CANV-01>

    Refs: phase 106 TEST-04"
    ```
    The literal string `PHASE-106-DELETE:` is MANDATORY in any commit that deletes test files, so the audit can be surfaced via:
    ```bash
    git log --grep='PHASE-106-DELETE:' --all
    ```

    **E. Re-run affected module:**
    ```bash
    uv run pytest tests/<affected_module>/ --tb=short
    ```
    Must exit 0 before moving to the next batch.

    **F. D-10 skip/xfail audit (run once near the end):**
    ```bash
    grep -rn 'pytest.mark.skip\|pytest.mark.xfail\|@pytest.skip' tests/
    ```
    For every hit, check the line(s) immediately above for a justification comment. If absent, EITHER add a comment explaining why the skip exists (external service, covers removed feature, etc.) OR remove the skip and fix/delete the test. Commit any additions in a final "docs(pytest): annotate surviving skips" commit referencing D-10.

    **G. Final full-suite gate for this plan:**
    ```bash
    uv run ruff check .
    uv run ruff format --check .
    uv run pytest
    ```
    Must exit 0. If not, either another failure surfaced (triage it the same way) or a fix rippled (pitfall 3 — revert and try a per-test fix). Do NOT proceed until `uv run pytest` is 0.
