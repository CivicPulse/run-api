<objective>
Repair the runtime orchestration gap by enforcing bounded primary chunk concurrency with durable successor promotion while consuming the file-size-aware planner from Plan 65-01.

Purpose: Phase 60 currently creates all chunks then defers every chunk immediately, which leaves `import_max_chunks_per_import` unused and causes the audit failures `INT-01` and `FLOW-02`. This plan converts that eager fan-out into a rolling window while preserving the existing chunk table, parent entrypoint, and secondary-task lifecycle from Phases 60-63.
Output: `process_import` schedules only the first capped chunk set, `process_import_chunk` promotes the next pending chunk when its primary range work is done, and regression coverage proves capped end-to-end dispatch.
