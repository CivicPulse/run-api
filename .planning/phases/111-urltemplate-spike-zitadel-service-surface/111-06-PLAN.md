<action>
**Pre-phase baseline capture (added per review W3).**

Capture the LAST row of `web/e2e-runs.jsonl` as it stood BEFORE Phase 111's work began, and persist it to `artifacts/pre-phase-baseline.txt` so T3's "compare against pre-phase pass counts" step has an authoritative reference. Without this snapshot, T3 would compare Run 1 against itself (the last row IS Run 1 at that moment — a tautology).

Run:
```bash
# Find the last e2e-runs.jsonl row authored before Phase 111's first commit.
# Phase 111's first commit is identified by the earliest commit touching any
# file under .planning/phases/111-urltemplate-spike-zitadel-service-surface/
# or web/e2e/invite-urltemplate-spike.spec.ts.
FIRST_P111_COMMIT=$(git log --reverse --format=%H -- \
  .planning/phases/111-urltemplate-spike-zitadel-service-surface/ \
  web/e2e/invite-urltemplate-spike.spec.ts \
  2>/dev/null | head -1)

if [ -n "$FIRST_P111_COMMIT" ]; then
  # Dump e2e-runs.jsonl AS IT WAS at the commit just before Phase 111 started.
  PARENT=$(git rev-parse "${FIRST_P111_COMMIT}^" 2>/dev/null || echo HEAD)
  git show "${PARENT}:web/e2e-runs.jsonl" 2>/dev/null | tail -1 \
    > .planning/phases/111-urltemplate-spike-zitadel-service-surface/artifacts/pre-phase-baseline.txt || true
fi

# Fallback: if the above produced nothing (e2e-runs.jsonl didn't exist yet, or
# the phase's first commit touched no e2e files), capture the CURRENT last row
# BEFORE running any Phase 111 tests — take the snapshot NOW, before T1/T2/T3
# append their runs. Do this early in Plan 06 before T1 runs.
if [ ! -s .planning/phases/111-urltemplate-spike-zitadel-service-surface/artifacts/pre-phase-baseline.txt ]; then
  tail -1 web/e2e-runs.jsonl \
    > .planning/phases/111-urltemplate-spike-zitadel-service-surface/artifacts/pre-phase-baseline.txt 2>/dev/null \
    || echo '{"note":"e2e-runs.jsonl empty or missing at baseline capture time"}' \
    > .planning/phases/111-urltemplate-spike-zitadel-service-surface/artifacts/pre-phase-baseline.txt
fi
```

Record the baseline row's `pass`, `fail`, `skip`, `total` values in `111-BASELINE.md`'s "Pre-phase baseline" section (T5 consumes this).

**This task MUST run before T1-T3.** T1/T2/T3 append to `web/e2e-runs.jsonl`; if T0 runs after T3, the "last row" captured is Phase 111's Run 1 — same tautology the review flagged.
</action>
