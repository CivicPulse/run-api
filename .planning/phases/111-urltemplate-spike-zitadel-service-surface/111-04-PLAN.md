    async def create_invite_code(
        self,
        user_id: str,
        url_template: str,
    ) -> str:
        """Create a one-time invite code for a ZITADEL user.

        POSTs to ``/v2/users/{userId}/invite_code`` with
        ``{"returnCode": {}, "urlTemplate": url_template}``. The
        ``returnCode`` knob tells ZITADEL to return the code in the
        response body instead of mailing it; CivicPulse emails the code
        via Mailgun so DMARC/SPF stays under our control.

        Args:
            user_id: ZITADEL user ID (must already exist).
            url_template: Full URL template with ZITADEL placeholders.
                Example:
                ``https://run.civpulse.org/invites/<token>?zitadelCode={{.Code}}&userID={{.UserID}}``

        Returns:
            The invite code as a string. Pass it as the ``code`` query
            parameter of the accept-invitation URL.

        Raises:
            ZitadelUnavailableError: Retry exhaustion.
            httpx.HTTPStatusError: 4xx responses (e.g. 404 if user
                unknown; 400 if url_template is invalid).
        """
        async def _post() -> dict:
            token = await self._get_token()
            headers = self._auth_headers(token)
            async with httpx.AsyncClient(
                verify=self._verify_tls, timeout=10.0
            ) as client:
                response = await client.post(
                    f"{self.base_url}/v2/users/{user_id}/invite_code",
                    headers=headers,
                    json={"returnCode": {}, "urlTemplate": url_template},
                )
                response.raise_for_status()
                return response.json()

        data = await self._retry_on_5xx(_post)
        # ZITADEL v2 response shape: the code is under inviteCode.code OR
        # directly under "code" depending on API version. Try the nested
        # shape first (current v2 docs); fall back to top-level.
        invite_code_obj = data.get("inviteCode")
        if isinstance(invite_code_obj, dict) and "code" in invite_code_obj:
            code = invite_code_obj["code"]
        elif "code" in data:
            code = data["code"]
        else:
            raise ZitadelUnavailableError(
                f"ZITADEL invite_code response missing code field: keys={list(data.keys())}"
            )
        logger.debug("ZITADEL invite code issued for user {}", user_id)
        return code
</action>

<threat_model>
**STRIDE:**
- **Spoofing (MEDIUM):** Invite codes are bearer credentials. Mitigation: single-use (ZITADEL consumes on accept); emitted only to the caller (logged method call, not logged code value); delivered via Mailgun under our SPF/DMARC.
- **Tampering (LOW):** Codes are opaque; ZITADEL validates server-side.
- **Information Disclosure (HIGH if code is logged, MITIGATED):** Mitigation: `logger.debug` states "code issued for user {user_id}" without including the code. Unit test T2 asserts the code is never in log output.
- **DoS (LOW):** One POST per call; bounded retry; rate-limit head-room.
- **Elevation of Privilege (MITIGATED by Plan 05):** Service-account scope; same mitigation as Plan 03.
- **Repudiation (LOW):** `logger.debug` with user_id provides a trail.

**Blocking severity: none** — HIGH information-disclosure risk is explicitly mitigated and tested.
</threat_model>
