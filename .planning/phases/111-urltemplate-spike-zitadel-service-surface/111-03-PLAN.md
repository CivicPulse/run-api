    async def ensure_human_user(
        self,
        email: str,
        given_name: str,
        family_name: str,
        org_id: str,
    ) -> EnsureHumanUserResult:
        """Ensure a human user with the given email exists (idempotent).

        Searches by email first; returns the existing user with
        ``created=False`` if found. Otherwise creates the user with
        ``email.isVerified=true`` and returns ``created=True``.

        The search-first ordering inverts ``ensure_project_grant`` because
        this method's contract requires a reliable ``created`` flag. On a
        create race (409 after an initial miss), re-searches and returns
        the winning user with ``created=False``.

        Args:
            email: User email address. Lowercased before the search and
                before being passed to the create endpoint.
            given_name: First name (profile.givenName).
            family_name: Last name (profile.familyName).
            org_id: ZITADEL org ID; set as the ``x-zitadel-orgid`` header.

        Returns:
            EnsureHumanUserResult with the ZITADEL user ID and a
            ``created`` flag distinguishing new vs existing identities.

        Raises:
            ZitadelUnavailableError: Retry exhaustion on either the search
                or the create call.
        """
        normalized = email.lower()

        async def _search() -> str | None:
            token = await self._get_token()
            headers = self._auth_headers(token)
            headers["x-zitadel-orgid"] = org_id
            async with httpx.AsyncClient(
                verify=self._verify_tls, timeout=10.0
            ) as client:
                response = await client.post(
                    f"{self.base_url}/v2/users",
                    headers=headers,
                    json={
                        "queries": [
                            {
                                "emailQuery": {
                                    "emailAddress": normalized,
                                    "method": "TEXT_QUERY_METHOD_EQUALS",
                                }
                            }
                        ],
                    },
                )
                response.raise_for_status()
                results = response.json().get("result", [])
                if results:
                    return results[0]["userId"]
                return None

        async def _create() -> str:
            token = await self._get_token()
            headers = self._auth_headers(token)
            headers["x-zitadel-orgid"] = org_id
            async with httpx.AsyncClient(
                verify=self._verify_tls, timeout=10.0
            ) as client:
                response = await client.post(
                    f"{self.base_url}/v2/users/human",
                    headers=headers,
                    json={
                        "username": normalized,
                        "profile": {
                            "givenName": given_name,
                            "familyName": family_name,
                        },
                        "email": {
                            "email": normalized,
                            "isVerified": True,
                        },
                    },
                )
                response.raise_for_status()
                return response.json()["userId"]

        # Search first (D-EHU-01)
        existing_id = await self._retry_on_5xx(_search)
        if existing_id is not None:
            logger.debug(
                "ZITADEL user already exists for {}: {}", normalized, existing_id
            )
            return EnsureHumanUserResult(user_id=existing_id, created=False)

        # Miss -> create
        try:
            new_id = await self._retry_on_5xx(_create)
            logger.info("ZITADEL user created for {}: {}", normalized, new_id)
            return EnsureHumanUserResult(user_id=new_id, created=True)
        except httpx.HTTPStatusError as exc:
            if exc.response.status_code != 409:
                raise
            # Lost a create race; re-search
            logger.debug(
                "ZITADEL user create 409 for {}; re-searching", normalized
            )
            final_id = await self._retry_on_5xx(_search)
            if final_id is None:
                # Should be unreachable; 409 without a search hit is pathological
                raise ZitadelUnavailableError(
                    f"ZITADEL 409 on create but no user found for {normalized}"
                ) from exc
            return EnsureHumanUserResult(user_id=final_id, created=False)

Note the request body keys match ZITADEL v2 User Service docs (see canonical_refs §ZITADEL External Docs). If the exact field names (`emailQuery`, `userId`, `username`) differ when hit live in T4, update them against the actual response shape — documented behavior wins over documented field names.

<threat_model>
**STRIDE:**
- **Spoofing (MEDIUM):** Creating user identities is a privileged operation. Mitigation: Runs under the runtime ZITADEL service-account scoped to the CivicPulse org (Plan 05 narrows it). Emails and IDs are never trusted as input from end-users; callers supply them after their own validation (Phase 113 invite flow).
- **Tampering (LOW):** All writes are idempotent. Mitigation: search-first-then-create returns existing users unchanged; no update path in this method.
- **Repudiation (LOW):** `logger.info` on create provides an audit trail. Mitigation: Loguru defaults to structured logs; timestamp is UTC.
- **Information Disclosure (MEDIUM):** Email addresses flow through logs. Mitigation: logs the lowercase email (no PII beyond what's already on file); org_id is the only other identifier. Consider hashing emails in prod (non-blocking, tracked elsewhere).
- **DoS (LOW):** Each call is 1-3 ZITADEL requests; rate-limit head-room is large (50 req/s). Retry is bounded at 3 attempts.
- **Elevation of Privilege (HIGH risk if scope too broad, MITIGATED by Plan 05):** The service-account identity must be scoped narrowly; if it holds `IAM_OWNER`, it can escalate. Mitigation: Plan 05 audits and narrows per SEC-03. Ship this method AFTER or alongside Plan 05's narrowing.

**Blocking severity: none** — EoP is addressed by Plan 05 in the same phase.
</threat_model>
