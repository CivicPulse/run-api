<task id="111-02-T2" type="execute">

Add the `_retry_on_5xx` method to `ZitadelService`, placed directly below `_auth_headers` (around line 101) so all new helpers cluster with auth/transport plumbing. Exact implementation:

```python
async def _retry_on_5xx(
    self,
    fn: Callable[[], Awaitable[T]],
    *,
    attempts: int = 3,
    backoff: tuple[float, ...] = (1.0, 2.0, 4.0),
) -> T:
    """Invoke `fn` with bounded exponential retry on transient errors.

    Retries only on `httpx.ConnectError`, `httpx.TimeoutException`, and
    `httpx.HTTPStatusError` where `response.status_code >= 500`. All other
    exceptions (4xx, validation, auth) propagate immediately.

    Args:
        fn: Zero-arg async callable. Each attempt re-invokes it, so callers
            that open an `httpx.AsyncClient` must do so *inside* `fn`, not
            once around the retry call (otherwise attempt 2 uses a closed
            client).
        attempts: Max total attempts (default 3).
        backoff: Sleep seconds BETWEEN attempts. Must have length
            `attempts - 1`; index `i` is the sleep before attempt `i + 2`.

    Returns:
        Whatever `fn` returns on a successful attempt.

    Raises:
        ZitadelUnavailableError: On exhaustion (all attempts failed with
            a retry-eligible error), chained from the last captured
            exception.
        Any non-retry-eligible exception: Propagated immediately from the
            first attempt that raises it.
    """
    last_exc: Exception | None = None
    for attempt in range(1, attempts + 1):
        try:
            return await fn()
        except (httpx.ConnectError, httpx.TimeoutException) as exc:
            last_exc = exc
            logger.debug(
                "ZitadelService retry: transient transport error on attempt {}/{}: {}",
                attempt,
                attempts,
                exc,
            )
        except httpx.HTTPStatusError as exc:
            if exc.response.status_code < 500:
                raise
            last_exc = exc
            logger.debug(
                "ZitadelService retry: HTTP {} on attempt {}/{}",
                exc.response.status_code,
                attempt,
                attempts,
            )
        if attempt < attempts:
            await asyncio.sleep(backoff[attempt - 1])

    raise ZitadelUnavailableError(
        f"ZITADEL unavailable after {attempts} attempts"
    ) from last_exc
```

Place after the `_auth_headers` method definition (currently ending near line 101). Keep existing methods untouched.
