1. **app/models/voter.py** — Voter model with all canonical fields exactly as specified in research:

   - Address: address_line1, address_line2, city, state, zip_code, county

   - Each with: id (UUID PK), campaign_id (FK campaigns.id), voter_id (FK voters.id), value (String), type (String 20: home/work/cell for phone, home/work for email, home/work/mailing for address), is_primary (Boolean default False), source (String 50: "import" or "manual")
   - VoterAddress has: address_line1, address_line2, city, state, zip_code instead of single value field
   - Timestamps: created_at, updated_at

3. **app/models/voter_list.py** — VoterList model:
   - id (UUID PK), campaign_id (FK), name (String 255), description (Text, nullable), list_type (StrEnum: "static"/"dynamic"), filter_query (JSONB, nullable — used for dynamic lists), created_by (FK users.id), created_at, updated_at
   - VoterListMember join table: voter_list_id + voter_id composite PK, added_at timestamp

4. **app/models/voter_interaction.py** — VoterInteraction model:
   - Exactly as in research: id, campaign_id, voter_id, type (InteractionType StrEnum with note/tag_added/tag_removed/import/contact_updated), payload (JSONB), created_by (FK users.id as String), created_at
   - Use native_enum=False in Enum() for extensibility via migrations

5. **app/models/import_job.py** — ImportJob and FieldMappingTemplate:
   - ImportJob exactly as research: id, campaign_id, status (ImportStatus StrEnum: pending/uploaded/queued/processing/completed/failed), file_key, original_filename, source_type, field_mapping (JSONB), detected_columns (JSONB), suggested_mapping (JSONB), total_rows, imported_rows, skipped_rows, error_report_key, error_message, created_by, created_at, updated_at
   - FieldMappingTemplate: id (UUID PK), campaign_id (FK, nullable — NULL for system templates like L2), name (String 255), source_type (String 50), mapping (JSONB), is_system (Boolean default False), created_by (FK users.id, nullable), created_at
