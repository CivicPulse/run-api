```python
def apply_date_filter(query, column, start_date: date | None, end_date: date | None):
    if start_date:
        query = query.where(column >= datetime.combine(start_date, time.min))
    if end_date:
        query = query.where(column <= datetime.combine(end_date, time.max))
    return query
```

Create `app/services/dashboard/` package with three service modules. Each service class follows the existing project pattern: stateless class with async methods that accept `session: AsyncSession`, `campaign_id: uuid.UUID`, and optional `start_date`/`end_date`.

**`app/services/dashboard/__init__.py`:**
Export all three service classes.

**`app/services/dashboard/canvassing.py` -- CanvassingDashboardService:**

Methods:
1. `get_summary(session, campaign_id, start_date=None, end_date=None) -> CanvassingSummary`
   - Query `VoterInteraction` WHERE `type == DOOR_KNOCK` and `campaign_id` matches
   - COUNT total as `doors_knocked`
   - COUNT WHERE `payload['result_code'].astext` IN ('supporter', 'undecided', 'opposed', 'refused') as `contacts_made`
   - COUNT each DoorKnockResult value individually for OutcomeBreakdown using `.filter()` on `func.count()`
   - Calculate `contact_rate = contacts_made / doors_knocked` (guard division by zero)
   - Apply `apply_date_filter` on `VoterInteraction.created_at`

2. `get_by_turf(session, campaign_id, start_date=None, end_date=None, turf_id=None, cursor=None, limit=20) -> list[TurfBreakdown]`
   - JOIN `VoterInteraction` -> `WalkList` via `CAST(payload['walk_list_id'].astext, UUID)` -> `Turf` via `WalkList.turf_id`
   - GROUP BY `Turf.id, Turf.name`
   - Same count logic as summary but per-turf
   - Optional `turf_id` filter for single-turf view
   - Cursor pagination on `Turf.id` (UUID-based cursor)

3. `get_by_canvasser(session, campaign_id, start_date=None, end_date=None, cursor=None, limit=20) -> list[CanvasserBreakdown]`
   - Query `VoterInteraction` WHERE `type == DOOR_KNOCK`, GROUP BY `created_by`
   - LEFT JOIN `User` on `VoterInteraction.created_by == User.id` for `display_name`
   - Same count logic per-canvasser
   - Cursor pagination on `User.id`

**CRITICAL -- JSONB access pattern:** Use `VoterInteraction.payload["result_code"].astext` for string comparison against DoorKnockResult values. Use `VoterInteraction.payload["walk_list_id"].astext.cast(PgUUID)` for the join to WalkList.

**`app/services/dashboard/phone_banking.py` -- PhoneBankingDashboardService:**

Methods:
1. `get_summary(session, campaign_id, start_date=None, end_date=None) -> PhoneBankingSummary`
   - Query `VoterInteraction` WHERE `type == PHONE_CALL` and `campaign_id` matches
   - COUNT total as `calls_made`
   - COUNT WHERE `payload['result_code'].astext` IN ('answered', 'refused') as `contacts_reached`
   - COUNT each CallResultCode value for CallOutcomeBreakdown
   - Calculate `contact_rate = contacts_reached / calls_made`

2. `get_by_session(session, campaign_id, start_date=None, end_date=None, session_id=None, cursor=None, limit=20) -> list[SessionBreakdown]`
   - JOIN `VoterInteraction` -> `PhoneBankSession` via `CAST(payload['session_id'].astext, UUID)`
   - GROUP BY session fields, include session status
   - Cursor pagination on `PhoneBankSession.id`

3. `get_by_caller(session, campaign_id, start_date=None, end_date=None, cursor=None, limit=20) -> list[CallerBreakdown]`
   - Query `VoterInteraction` WHERE `type == PHONE_CALL`, GROUP BY `created_by`
   - LEFT JOIN `User` for display_name
   - Cursor pagination on `User.id`

4. `get_by_call_list(session, campaign_id, start_date=None, end_date=None, cursor=None, limit=20) -> list[CallListBreakdown]`
   - JOIN `VoterInteraction` -> `CallList` via `CAST(payload['call_list_id'].astext, UUID)`
   - SELECT call_list name, total_entries, completed_entries from `CallList` columns
   - COUNT interactions for calls_made, contacts_reached
   - Calculate completion_rate = completed_entries / total_entries
   - Cursor pagination on `CallList.id`

**`app/services/dashboard/volunteer.py` -- VolunteerDashboardService:**

Methods:
1. `get_summary(session, campaign_id, start_date=None, end_date=None) -> VolunteerSummary`
   - COUNT `Volunteer` WHERE `status == ACTIVE` as `active_volunteers`
   - COUNT total `Volunteer` as `total_volunteers`
   - COUNT `Shift` WHERE `status == SCHEDULED` as `scheduled_shifts`
   - COUNT `Shift` WHERE `status == COMPLETED` as `completed_shifts`
   - SUM hours from `ShiftVolunteer` WHERE `check_out_at IS NOT NULL`: use `func.coalesce(adjusted_hours, extract('epoch', check_out_at - check_in_at) / 3600.0)` as `total_hours`
   - Date filter applies to `Shift.start_at` for shift counts and `ShiftVolunteer.check_in_at` for hours

2. `get_by_volunteer(session, campaign_id, start_date=None, end_date=None, cursor=None, limit=20) -> list[VolunteerBreakdown]`
   - JOIN `Volunteer` -> `ShiftVolunteer` -> `Shift`
   - GROUP BY volunteer fields
   - COUNT shifts where ShiftVolunteer.status == CHECKED_OUT as shifts_completed
   - SUM hours using same coalesce pattern
   - Cursor pagination on `Volunteer.id`

3. `get_by_shift(session, campaign_id, start_date=None, end_date=None, shift_type=None, cursor=None, limit=20) -> list[ShiftBreakdown]`
   - LEFT JOIN `Shift` -> `ShiftVolunteer`
   - GROUP BY shift fields
   - COUNT signed_up (status != CANCELLED), checked_in (status == CHECKED_IN or CHECKED_OUT), checked_out (status == CHECKED_OUT)
   - Optional `shift_type` filter (ShiftType enum)
   - Cursor pagination on `Shift.id`
