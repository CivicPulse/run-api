    Helper: `isNetworkError(err: unknown): boolean`
    - Returns true if err instanceof TypeError (ky throws TypeError for network failures)

    Helper: `isConflict(err: unknown): boolean`
    - Check if err has .response.status === 409 (ky HTTPError pattern)

    Helper: `async replayMutation(item: QueueItem): Promise<void>`
    - switch on item.type:
      - "door_knock": POST to `api/v1/campaigns/${item.campaignId}/walk-lists/${item.resourceId}/door-knocks` with json: item.payload
      - "call_record": POST to `api/v1/campaigns/${item.campaignId}/phone-bank-sessions/${item.resourceId}/calls` with json: item.payload
    - Use api.post().json() pattern from existing codebase

    Main export: `function useSyncEngine(): void`

    Inside the hook:
    - const isOnline = useConnectivityStatus()
    - const queryClient = useQueryClient()
    - const drainRef = useRef for stable drain function reference

    Define `drainQueue` async function:
    1. const { items, isSyncing } = useOfflineQueueStore.getState()
    2. if (items.length === 0 || isSyncing || !navigator.onLine) return
    3. useOfflineQueueStore.getState().setSyncing(true)
    4. const syncedCampaignIds = new Set<string>() -- track for invalidation
    5. const syncedResourceIds = new Map<string, { type: string, campaignId: string }>()
    6. Loop through [...items] (copy to avoid mutation during iteration):
       - try: await replayMutation(item)
         - useOfflineQueueStore.getState().remove(item.id)
         - syncedCampaignIds.add(item.campaignId)
         - syncedResourceIds.set(item.resourceId, { type: item.type, campaignId: item.campaignId })
       - catch (err):
         - if isConflict(err): remove item silently, continue
         - else if item.retryCount >= 2: incrementRetry(item.id), continue (skip, will retry next drain)
         - else: incrementRetry(item.id), break (stop drain, retry later)
    7. useOfflineQueueStore.getState().setSyncing(false)
    8. Post-sync: if syncedResourceIds.size > 0:
       - For each synced resource, invalidate appropriate queries:
         - door_knock type: invalidateQueries({ queryKey: ["walk-list-entries-enriched", campaignId, resourceId] })
         - call_record type: invalidateQueries({ queryKey: ["campaigns", campaignId, "phone-bank-sessions", resourceId] })
       - If queue is now empty (useOfflineQueueStore.getState().items.length === 0):
         - toast.success("All caught up!")
       - Check for team progress (entries completed by others):
         - After invalidation completes, compare pre/post entry data for door_knock resources
         - This is best-effort: get query data before invalidation (snapshot), then after, count new "visited" entries not in our synced set
         - If othersCount > 0: toast(`${othersCount} doors were visited by others`)

    useEffect for online event trigger:
    - When isOnline transitions to true, debounce by 1000ms, then call drainQueue()
    - Use setTimeout with 1000ms delay (connectivity debounce per UI-SPEC.md)
    - Clean up timeout on unmount or re-trigger

    useEffect for periodic drain (30-second interval):
    - setInterval(() => drainQueue(), 30_000) when online
    - Clear interval on unmount or when offline
    - Guard: only run interval when isOnline === true

    ```typescript
    const payload = {

    doorKnockMutation.mutate(payload, {
      onError: (err) => {
        // Network error: queue for offline sync instead of reverting
        if (err instanceof TypeError) {
          useOfflineQueueStore.getState().push({

        }
      },
    })
    ```

    CRITICAL: The .mutate() call-site onError overrides the mutation-level onError defined in useDoorKnockMutation. This means when we pass onError here, the revertOutcome() in useCanvassing.ts line 30-33 does NOT run. This is exactly what we want for offline queueing.
