2. Create `alembic/versions/003_canvassing_operations.py` with manual migration (do NOT use autogenerate):
   - Revision ID: generate a 12-char hex string
   - Down revision: the voter data models migration revision ID (read from 002_voter_data_models.py)

   upgrade():
   a. Enable PostGIS: `op.execute("CREATE EXTENSION IF NOT EXISTS postgis")`
   b. Add geom column to voters: `op.add_column("voters", sa.Column("geom", Geometry("POINT", srid=4326), nullable=True))`
   c. Create GiST spatial index on voters.geom: `op.create_index("ix_voters_geom", "voters", ["geom"], postgresql_using="gist")`
   d. Backfill voter geom from lat/long:
      ```sql
      UPDATE voters SET geom = ST_SetSRID(ST_MakePoint(longitude, latitude), 4326)
      WHERE latitude IS NOT NULL AND longitude IS NOT NULL AND geom IS NULL
      ```
   e. Create `turfs` table: id (UUID PK), campaign_id (UUID FK NOT NULL), name (VARCHAR 255 NOT NULL), description (TEXT nullable), status (VARCHAR 50 NOT NULL DEFAULT 'draft'), boundary (Geometry POLYGON 4326 NOT NULL), created_by (VARCHAR FK users.id NOT NULL), created_at (TIMESTAMP DEFAULT now()), updated_at (TIMESTAMP DEFAULT now())
      - GiST spatial index on boundary (automatic from geoalchemy2 spatial_index=True, but verify or create explicitly)
      - Index on (campaign_id, status)
   f. Create `walk_lists` table: id, campaign_id, turf_id (FK turfs.id), voter_list_id (FK voter_lists.id, nullable), script_id (FK survey_scripts.id, nullable), name, total_entries (INT DEFAULT 0), visited_entries (INT DEFAULT 0), created_by (FK users.id), created_at
      - Index on campaign_id
   g. Create `walk_list_entries` table: id, walk_list_id (FK walk_lists.id), voter_id (FK voters.id), household_key (VARCHAR 500 nullable), sequence (INT NOT NULL), status (VARCHAR 50 DEFAULT 'pending')
      - Index on (walk_list_id, sequence)
   h. Create `walk_list_canvassers` table: walk_list_id (FK PK), user_id (FK PK), assigned_at (TIMESTAMP DEFAULT now())
   i. Create `survey_scripts` table: id, campaign_id, title (VARCHAR 255), description (TEXT nullable), status (VARCHAR 50 DEFAULT 'draft'), created_by (FK users.id), created_at, updated_at
   j. Create `survey_questions` table: id, script_id (FK survey_scripts.id), position (INT NOT NULL), question_text (TEXT NOT NULL), question_type (VARCHAR 50 NOT NULL), options (JSONB nullable)
      - Index on (script_id, position)
   k. Create `survey_responses` table: id, campaign_id, script_id (FK), question_id (FK), voter_id (FK), answer_value (TEXT NOT NULL), answered_by (FK users.id), answered_at (TIMESTAMP DEFAULT now())
      - Index on (campaign_id, voter_id, script_id)

   l. RLS policies (follow exact pattern from 001/002 migrations):
      For each table with campaign_id (turfs, walk_lists, survey_scripts, survey_responses):
      ```sql
      ALTER TABLE {table} ENABLE ROW LEVEL SECURITY;
      ALTER TABLE {table} FORCE ROW LEVEL SECURITY;
      CREATE POLICY {table}_isolation ON {table}
        USING (campaign_id = current_setting('app.current_campaign_id', true)::uuid);
      GRANT SELECT, INSERT, UPDATE, DELETE ON {table} TO app_user;
      ```

      For tables without direct campaign_id (walk_list_entries, walk_list_canvassers, survey_questions):
      Use subquery through parent table, e.g.:
      ```sql
      CREATE POLICY walk_list_entries_isolation ON walk_list_entries
        USING (walk_list_id IN (SELECT id FROM walk_lists WHERE campaign_id = current_setting('app.current_campaign_id', true)::uuid));
      ```
      For survey_questions:
      ```sql
      CREATE POLICY survey_questions_isolation ON survey_questions
        USING (script_id IN (SELECT id FROM survey_scripts WHERE campaign_id = current_setting('app.current_campaign_id', true)::uuid));
      ```

   downgrade(): Drop tables in reverse order, drop geom column and index from voters. Do NOT drop PostGIS extension (other things may use it).

3. IMPORTANT: The walk_lists table has FK to survey_scripts, so survey_scripts must be created BEFORE walk_lists in the migration.
