  <name>Task 1: Implement TurfService and WalkListService with spatial queries and household clustering</name>
  <files>app/services/turf.py, app/services/walk_list.py, tests/unit/test_walk_lists.py</files>
  <action>
1. Create `app/services/turf.py` with TurfService:
   - `create_turf(session, campaign_id, data: TurfCreate, user_id) -> Turf`: Validate GeoJSON boundary using shapely: `geom = shape(data.boundary)`, check `geom.geom_type == "Polygon"`, check `geom.is_valid` (raise ValueError with `explain_validity()` if not), check `geom.area > 0` (reject degenerate). Convert via `from_shape(geom, srid=4326)`. Create Turf row.
   - `get_turf(session, turf_id) -> Turf | None`: Simple get by ID
   - `list_turfs(session, campaign_id, status_filter=None, cursor=None, limit=20) -> tuple[list[Turf], str | None, bool]`: Paginated list with optional status filter, cursor-based pagination (created_at|id)
   - `update_turf(session, turf_id, data: TurfUpdate) -> Turf`: Update fields. If boundary changed, re-validate. Only allow status transitions: draft->active, active->completed, draft->active->completed (no backward)
   - `delete_turf(session, turf_id) -> None`: Delete turf (CASCADE handles walk lists)
   - `get_voter_count(session, turf_id) -> int`: Count voters whose geom is within the turf boundary using `func.ST_Contains(Turf.boundary, Voter.geom)`. Exclude voters with NULL geom.
   - Helper `_wkb_to_geojson(wkb: WKBElement) -> dict`: Convert WKBElement to GeoJSON dict using `to_shape()` then `mapping()`

2. Create `app/services/walk_list.py` with WalkListService:
   - `generate_walk_list(session, campaign_id, data: WalkListCreate, user_id) -> WalkList`:
     a. Verify turf exists and belongs to campaign
     b. Build spatial query: `select(Voter).where(func.ST_Contains(subquery_turf_boundary, Voter.geom))` where subquery gets boundary from Turf by turf_id
     c. If voter_list_id provided, join with VoterListMember to intersect with target universe
     d. Exclude voters with NULL geom
     e. Fetch all matching voters
     f. Sort voters using `parse_address_sort_key()`: extract (street_name, house_number) from address_line1 via regex `^(\d+)\s+(.+)$`, sort by (street_name.upper(), house_number, last_name)
     g. Cluster by household: group consecutive voters with same normalized `(address_line1.upper().strip(), zip_code[:5])`. Use `household_id` from voter if available, else address-based key
     h. Create WalkListEntry for each voter with: sequence number (incrementing across households), household_key (the normalized address tuple as string), status=PENDING
     i. Set walk_list.total_entries = count of entries created
     j. Return the walk list with stats. Include `voters_excluded_no_geom` count in response metadata if any voters were excluded
   - `get_walk_list(session, walk_list_id) -> WalkList | None`
   - `list_walk_lists(session, campaign_id, turf_id=None, cursor=None, limit=20) -> tuple[list, str|None, bool]`: Paginated, optional turf filter
   - `get_entries(session, walk_list_id, status_filter=None, cursor=None, limit=50) -> tuple[list[WalkListEntry], str|None, bool]`: Ordered by sequence, optional status filter
   - `update_entry_status(session, entry_id, status: WalkListEntryStatus) -> WalkListEntry`: Update individual entry status (used for "skipped")
   - `assign_canvasser(session, walk_list_id, user_id) -> WalkListCanvasser`: Add canvasser to walk list
   - `remove_canvasser(session, walk_list_id, user_id) -> None`: Remove assignment
   - `list_canvassers(session, walk_list_id) -> list[WalkListCanvasser]`: List assigned canvassers
   - `delete_walk_list(session, walk_list_id) -> None`: Delete (CASCADE on entries and canvassers)

   Import `from app.models.voter_list import VoterListMember` for the optional voter list intersection.
   Use `from geoalchemy2 import func as geo_func` or `from sqlalchemy import func` with `func.ST_Contains`.

   - Implement `test_walk_list_is_frozen_snapshot`: verify WalkListEntry model has expected fields (walk_list_id, voter_id, sequence, status)
   - Implement `test_household_clustering`: unit test the address-based household key generation logic -- pass in a list of voter-like objects with addresses and verify clustering groups same-address voters
   - Implement `test_address_sort_order`: unit test the parse_address_sort_key function with sample addresses, verify street-name-then-number ordering
   - Implement `test_canvasser_assignment`: verify WalkListCanvasser model has walk_list_id and user_id PKs
   - Implement `test_entry_status_update`: verify WalkListEntryStatus enum has pending/visited/skipped values
   - Use mock_db pattern from existing tests where database interaction is needed. For pure logic (sorting, clustering), test the helper functions directly.
