## Typography

All values use the existing project tokens established in `web/src/index.css`.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px | 400 (regular) | 1.5 |
| Label | 14px | 400 (regular) | 1.4 |
| Heading | 20px | 600 (semibold) | 1.2 |
| Display | 28px | 700 (bold) | 1.1 |

Call duration timer: 16px, weight 600 (semibold), monospace (`font-mono`). Real-time updates must not cause layout shift — use fixed-width character display (`tabular-nums`).

Section labels within voter info panel use the established pattern from `call.tsx`: 12px, weight 500, `text-muted-foreground`, uppercase, tracked (`text-xs font-medium text-muted-foreground uppercase tracking-wide`).

## Color

All color values reference existing project CSS custom properties from `web/src/index.css`. No new color tokens are introduced in this phase.

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` `oklch(0.995 0.003 240)` | Page background, call status bar surface |
| Secondary (30%) | `--card` / `--secondary` `oklch(0.965 0.005 240)` | Call status bar background, mode badge background, voter info panel |
| Accent (10%) | `--primary` `oklch(0.37 0.12 240)` | Active call button (in-progress state), hang-up button hover ring, mute toggle active state |
| Destructive | `--destructive` `oklch(0.55 0.245 27.325)` | Hang-up button fill while a call is active, DNC block indicator |

Accent (`--primary`) is reserved for: the hang-up button background while a call is in-progress, the active state of the mute toggle, and the selected phone number button variant (inherits existing `variant="default"` pattern from `call.tsx`).

**Call mode badge colors:**
- "Browser Call" mode: `--status-info` background / `--status-info-foreground` text (`oklch(0.95 0.04 250)` / `oklch(0.4 0.1 250)`)
- "Phone" (tel: fallback) mode: `--status-neutral` background / `--status-neutral-foreground` text (`oklch(0.95 0 0)` / `oklch(0.4 0 0)`)

**DNC block state:** `--status-error` background / `--status-error-foreground` text on the DNC warning banner. Phone number text uses `text-muted-foreground` with strikethrough.

**Calling hours block state:** `--status-warning-muted` background (`oklch(0.96 0.04 85)`) for the calling hours banner. Text uses `--status-warning-foreground`.

## Interaction States

### Phone Row — Call Button

The `PhoneRow` component in `PhoneNumberList.tsx` evolves to support four states:

| State | Trigger | Visual |
|-------|---------|--------|
| Idle (browser call mode) | Twilio configured + WebRTC available | Button: `variant="outline"`, headphones icon + "Browser Call" text, CallModeIndicator badge "Browser Call" (info color) |
| Idle (tel: mode) | Twilio not configured OR WebRTC unavailable | Button: `variant="outline"`, phone icon + "Call" text, CallModeIndicator badge "Phone" (neutral color) — identical to current behavior |
| Calling (in-progress) | Browser call initiated | CallStatusBar appears below phone row: live timer + hang-up button (destructive) + mute toggle (outline/default toggle). Original call button disabled. |
| DNC blocked | Voter number is on DNC list | Call button replaced by `DncBlockBanner` inline message. No dial affordance. |
| Calling hours blocked | Current time outside allowed window | Call button is disabled. `CallingHoursBanner` appears above phone list showing allowed window and current time. |
| Microphone denied | `getUserMedia` permission rejected | Fall back silently to tel: mode. Toast: "Microphone access denied — using phone dialer instead." |

### Call Status Bar (in-progress only)

Appears inline below the active phone row. Does not navigate away or push content — it expands the row in place.

| Element | Spec |
|---------|------|
| Duration timer | `font-mono tabular-nums`, 16px semibold, updates every second |
| Hang-up button | `variant="destructive"`, size sm, min 44px touch target, PhoneOff icon + "Hang Up" label, aria-label="End call" |
| Mute toggle | `variant="outline"` when unmuted, `variant="default"` when muted, MicOff / Mic icon, size sm, min 44px touch target, aria-label="Mute call" / "Unmute call", aria-pressed state |
| Status label | Small badge: "Ringing" / "Connected" / "Ending..." — transitions as Twilio SDK device events fire |

### Post-Call Auto-Advance

When a browser call ends (hang-up or remote disconnect), auto-advance to the outcome recording step after a 500ms delay. No confirmation dialog. This is the same behavior as the existing tel: flow — outcome recording appears automatically.

## Accessibility Contract

All requirements enforce WCAG 2.1 AAA per the project design principles.

| Requirement | Specification |
|-------------|--------------|
| Touch targets | All interactive elements in phone row min 44×44px (existing `min-h-11 min-w-11` pattern from `PhoneNumberList.tsx`) |
| Focus management | When CallStatusBar appears, focus moves to the hang-up button automatically |
| Keyboard | Hang-up: focusable, Space/Enter activates. Mute toggle: Space/Enter toggles. Tab order: mute → hang-up → outcome buttons |
| Screen reader | Duration timer updates via `aria-live="polite"` region with 2-second debounce to avoid excessive announcements |
| Contrast | All text on status bar surface meets 7:1 for normal text. Hang-up button (destructive) meets 4.5:1 for large text |
| Reduced motion | Duration timer counter increments without CSS animation. Status badge transitions use `prefers-reduced-motion` guard — remove fade-in if reduced motion set |
| aria-pressed | Mute toggle uses `aria-pressed` boolean attribute |
| Role indicators | DNC and calling hours banners use `role="alert"` to announce blockage to screen readers immediately on render |

## Layout Contract

The call controls integrate into the existing `PhoneNumberList` / `PhoneRow` layout. No full-page overlays or modals are introduced — all call state is inline.

**Mobile (primary use case):**
- Single-column, full-width phone rows
- CallStatusBar expands below the active row — padding: 12px vertical, 16px horizontal
- Hang-up and mute buttons stack side-by-side, each taking 50% of available width minus gap
- DncBlockBanner and CallingHoursBanner are full-width inline banners with 12px padding

**Desktop (two-panel layout in `call.tsx`):**
- Phone rows remain in the left `VoterInfoPanel` (w-80 shrink-0)
- CallStatusBar appears within the same left panel below the active phone row
- No changes to the right panel (survey + outcomes)
