  <name>Task 1: Create dev entrypoint and add API service to Docker Compose</name>
  <files>scripts/dev-entrypoint.sh, docker-compose.yml</files>
  <action>
Create `scripts/dev-entrypoint.sh` (must be executable, chmod +x):

```bash
#!/bin/bash
set -e

echo "==> Running Alembic migrations..."
python -m alembic upgrade head

echo "==> Ensuring MinIO bucket exists..."
python -c "
import boto3, os
from botocore.exceptions import ClientError
s3 = boto3.client('s3',
    endpoint_url=os.environ.get('S3_ENDPOINT_URL', 'http://minio:9000'),
    aws_access_key_id=os.environ.get('S3_ACCESS_KEY_ID', 'minioadmin'),
    aws_secret_access_key=os.environ.get('S3_SECRET_ACCESS_KEY', 'minioadmin'),
    region_name='us-east-1')
bucket = os.environ.get('S3_BUCKET', 'voter-imports')
try:
    s3.head_bucket(Bucket=bucket)
    print(f'Bucket {bucket} already exists')
except ClientError:
    s3.create_bucket(Bucket=bucket)
    print(f'Created bucket {bucket}')
"

echo "==> Starting uvicorn with hot-reload..."
exec python -m uvicorn main:app --host 0.0.0.0 --port 8000 --reload --log-level info
```

Note: Use boto3 (not aioboto3) for the sync bucket creation â€” it's a one-shot call at startup. boto3 is already available as a transitive dependency of aioboto3 which is in the project deps.

Update `docker-compose.yml` to add the `api` service ABOVE the existing postgres/minio services. Key configuration:

- `build: .` (uses existing production Dockerfile)
- `command: bash /home/app/scripts/dev-entrypoint.sh` (overrides production CMD)
- `container_name: run-api-api`
- `ports: ["8000:8000"]`
- `env_file: .env`
- Environment overrides for container networking:
  - `DATABASE_URL=postgresql+asyncpg://postgres:postgres@postgres:5432/run_api`
  - `DATABASE_URL_SYNC=postgresql+psycopg2://postgres:postgres@postgres:5432/run_api`
  - `S3_ENDPOINT_URL=http://minio:9000`
- Volume mounts for hot-reload (bind mounts into /home/app/):
  - `./app:/home/app/app`
  - `./main.py:/home/app/main.py`
  - `./alembic:/home/app/alembic`
  - `./alembic.ini:/home/app/alembic.ini`
  - `./scripts:/home/app/scripts`
- `depends_on:` with `postgres: condition: service_healthy` and `minio: condition: service_healthy`
- `restart: unless-stopped`

Keep existing postgres, minio services and volumes unchanged.

Important: The Dockerfile runs as non-root `app` user. The bind-mounted scripts directory must be accessible. The entrypoint uses `bash` (available in python:3.13-slim). Alembic reads DATABASE_URL_SYNC from environment (overriding alembic.ini's sqlalchemy.url) via the env/run.py configuration already in place.
