- Per-step validation via `form.trigger(stepFields[step])` before advancing

**Step 1 - Campaign Details (per UI-SPEC):**
- Page heading: "Create Campaign"
- Fields: Name*, Type* (Select), Jurisdiction (Input), Election Date (Input type="date"), Candidate Name (Input)
- Required fields: name, type (marked with *)
- Button: "Continue to Review" (right-aligned)
- Back button: navigates to `/` (home)
- Validation: `form.trigger(["name", "type"])` before advancing

**Step 2 - Review (per UI-SPEC):**
- Heading: "Review your campaign"
- Summary card showing all entered values in a definition list format:
  - Name: {name}
  - Type: {type}
  - Jurisdiction: {jurisdiction_name || "Not specified"}
  - Election Date: {election_date || "Not specified"}
  - Candidate: {candidate_name || "Not specified"}
- Org confirmation line: "This campaign will be created in {currentOrg.name}." using `useOrgPermissions().currentOrg`
- Buttons: "Back" (goes to step 1), "Continue to Invite" (goes to step 3)

**Step 3 - Invite Team (per D-10 and UI-SPEC):**
- Heading: "Add team members"
- Body text: "Select existing org members to add to this campaign. You can always add more later."
- Fetch org members via `useOrgMembers()` hook
- Checkbox list of org members. Each row:
  - `<Checkbox>` + display_name + email (muted)
  - When checked, show `<Select>` dropdown for role assignment with options: viewer, volunteer, manager, admin
  - Default role: "viewer"
- Track selected members in local state: `useState<Map<string, string>>()` mapping user_id -> role
- Empty state: "No other members in your organization yet." when no other org members
- Filter out current user from the member list (they'll be auto-added as owner)
- Buttons:
  - "Back" (goes to step 2)
  - "Skip -- create without inviting" link (text-sm text-muted-foreground underline, calls submit with no invites)
  - "Create Campaign" primary button

**Submit logic:**
1. Create campaign via `api.post("api/v1/campaigns", { json: campaignData }).json<Campaign>()`
2. If team members selected, for each selected member call `addMemberToCampaign.mutateAsync({ campaignId: newCampaign.id, userId, role })`
3. On success: invalidate `["campaigns"]` and `["org", "campaigns"]` query keys, navigate to `/campaigns/${campaign.id}/dashboard`
4. On error: show error message "Failed to create campaign. Check your connection and try again." per UI-SPEC
5. Show `Loader2` spinner on submit button while pending

**Form guard:** Import and use `useFormGuard` if it exists in the project, otherwise add `beforeunload` listener when form is dirty: `form.formState.isDirty`.
