### Integration Defects (2 medium, 0 critical)

#### INT-01: Double FieldHeader Rendering
**Severity:** Medium | **Requirements:** NAV-03, NAV-04, TOUR-04
**Description:** Canvassing and phone-banking sub-routes each render their own `<FieldHeader>` in every conditional branch (5 instances each), duplicating the layout route's `<FieldHeader>`. Results in two stacked sticky headers. Sub-route headers lack `onHelpClick` (disabled help button).
**Impact:** Visual duplication only — back navigation and help replay work via layout header.
**Fix:** Remove `<FieldHeader>` from all sub-route render branches.

#### INT-02: Sync Engine Retry Eviction Off-by-One
**Severity:** Medium | **Requirement:** SYNC-02
**Description:** `useSyncEngine.ts:77` — when `retryCount >= 2`, the code calls `incrementRetry` then `continue` instead of `remove`. Items that permanently fail accumulate in localStorage forever.
**Impact:** Only affects permanently-unresolvable items. Normal sync works correctly.
**Fix:** Replace `incrementRetry(item.id)` with `remove(item.id)` on line 77.
