### Integration Issues (Non-Blocking)

**Issue 1: Double FieldHeader**
- **Files:** `canvassing.tsx` (5 render branches), `phone-banking.tsx` (5 render branches)
- **Effect:** Two stacked sticky headers in all canvassing and phone-banking states. Tour help button in layout header works; child headers' help buttons have no `onHelpClick`.
- **Affected requirements:** NAV-01, NAV-03, NAV-04 (functional, visual degradation)
- **Severity:** Tech Debt — requirements are functionally met, visual UX is degraded

**Issue 2: DoorListView MapPin missing hasAddress guard**
- **File:** `web/src/components/field/DoorListView.tsx` line 94
- **Effect:** MapPin `<a>` rendered unconditionally; `getGoogleMapsUrl` returns `destination=` (empty) for voters with null address fields. Produces broken Maps links.
- **Inconsistency:** `HouseholdCard.tsx` correctly uses `hasAddress()` to gate the Navigate button per Phase 36 intent.
- **Affected requirements:** CANV-05 (door list view), Phase 36 Maps consistency
- **Severity:** Tech Debt — non-breaking but inconsistent behavior
