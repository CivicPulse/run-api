### ⚠ Integration Defect: INT-02 — Double FieldHeader (High Severity)

**All 5 states** of canvassing and phone-banking render a second `FieldHeader` inside `<main>`, which is already wrapped by the layout route's `FieldHeader`.

```
$campaignId.tsx layout:
  <FieldHeader onHelpClick={handleHelpClick} />   ← layout header (active)
  <main>
    <Outlet />                                      ← sub-route renders ANOTHER FieldHeader here
  </main>
```

**Visible impact:**
- Two stacked navigation bars on canvassing and phone-banking screens
- Top header: active help button
- Sub-route headers: disabled help button (no `onHelpClick` prop passed)
- Two back arrows, two nav ARIA landmarks

**Fix:** Remove `<FieldHeader>` from all render branches in `canvassing.tsx` and `phone-banking.tsx`. The layout already provides the header.

**Affected requirements:** NAV-01, NAV-03, NAV-04, TOUR-04, A11Y-01
