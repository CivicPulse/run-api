## Integration Gaps

### INT-01: ZitadelService Not Initialized in App Lifespan (CRITICAL)

**File:** `app/main.py` lines 20-46
**Severity:** CRITICAL — runtime AttributeError

The lifespan function initializes `app.state.jwks_manager`, `app.state.storage_service`, and `app.state.broker`, but **never initializes `app.state.zitadel_service`**. Three route modules access it:

- `app/api/v1/campaigns.py` — campaign create and delete
- `app/api/v1/invites.py` — invite acceptance
- `app/api/v1/members.py` — role assignment, removal, ownership transfer

**Impact:** Campaign CRUD, invite acceptance, and member management fail at runtime with `AttributeError: 'State' object has no attribute 'zitadel_service'`.

**Affected requirements:** AUTH-02, AUTH-03, AUTH-05, AUTH-07

**Fix:** Add to lifespan in `app/main.py`:
```python
from app.services.zitadel import ZitadelService
app.state.zitadel_service = ZitadelService(
    issuer=settings.zitadel_issuer,
    client_id=settings.zitadel_service_client_id,
    client_secret=settings.zitadel_service_client_secret,
)
```

### INT-02: Phase 4 Models Missing from Alembic Discovery (MODERATE)

**File:** `app/db/base.py` lines 14-29
**Severity:** MODERATE — development-time issue, not runtime

Missing imports: `app.models.call_list`, `app.models.phone_bank`, `app.models.dnc`

Existing hand-written migration `004_phone_banking.py` works correctly. Only affects future `alembic revision --autogenerate` operations.
